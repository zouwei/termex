# Termex - 需求评审文档

> Terminal + Experience | 开源 AI 原生高颜值 SSH 客户端

---

## 1. 产品概述

### 1.1 产品定位

开源、AI 原生的高颜值 SSH 客户端，为开发者和运维工程师打造。

### 1.2 核心价值

| 价值支柱 | 描述 |
|----------|------|
| **Beautiful** | 现代 UI 设计，对标 Termius 颜值 |
| **Fast** | Rust + Tauri v2，原生级性能，非 Electron |
| **Intelligent** | AI 原生集成：危险命令拦截、自然语言转命令、智能补全 |
| **Free & Open** | MIT 协议，完全开源免费，配置自由迁移 |

### 1.3 目标用户

| 优先级 | 用户群 | 特征 |
|--------|--------|------|
| P0 | 独立开发者 / 个人 | 管理数台到数十台服务器，价格敏感，注重颜值 |
| P0 | 全栈工程师 | 日常高频 SSH，注重终端体验和效率 |
| P1 | 中小团队运维 | 需要配置共享和批量管理 |

## 2. 技术栈

```
┌──────────────────────────────────────────────┐
│                Termex App                     │
├─────────────────────┬────────────────────────┤
│   Frontend          │     Backend (Rust)      │
│                     │                         │
│  - Vue 3 + TS       │  - SSH (russh)          │
│  - xterm.js         │  - SFTP                 │
│  - Element Plus     │  - AES-256-GCM (ring)   │
│  - Tailwind CSS     │  - SQLite + SQLCipher    │
│                     │  - AI 接口层             │
│                     │  - 端口转发              │
├─────────────────────┴────────────────────────┤
│             Tauri v2 Runtime                  │
├──────────────────────────────────────────────┤
│          Win / Mac / Linux                    │
└──────────────────────────────────────────────┘
```

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Tauri v2 | Rust 后端天然适合 SSH/加密，比 Electron 轻 10x |
| 前端框架 | Vue 3 + Vite + TypeScript | 中文社区最强，已有 Tauri SSH 先例 |
| 终端渲染 | xterm.js | 业界标准，Tabby/VSCode 都在用 |
| SSH 库 | russh | 纯 Rust 实现，活跃维护，支持 SSH2 全特性 |
| 加密 | AES-256-GCM (ring) | Rust 生态最成熟的加密库 |
| 本地存储 | SQLite + SQLCipher | 加密数据库，成熟可靠 |
| UI 组件库 | Element Plus | 60-80+ 组件，中文文档完善 |
| CSS 框架 | Tailwind CSS | 原子化工具类，自定义样式灵活 |
| AI 集成 | HTTP 调用 LLM API | 轻量，不绑定特定 AI 提供商 |

## 3. 功能需求

### 3.1 V0.1 — MVP（核心可用）

#### F-001 SSH 连接管理
- **描述**：支持 SSH 连接的增删改查
- **字段**：名称、主机地址、端口、用户名、认证方式（密码/密钥）
- **优先级**：P0

#### F-002 服务器分组
- **描述**：支持将服务器按组管理，树形结构展示
- **支持**：创建组、拖拽排序、组内嵌套
- **优先级**：P0

#### F-003 密码加密存储
- **描述**：使用 AES-256-GCM 加密存储密码，密钥派生自用户主密码
- **安全要求**：密码不以明文形式存在于内存或磁盘
- **优先级**：P0

#### F-004 RSA 密钥认证
- **描述**：支持 RSA/Ed25519 密钥认证，支持导入和管理密钥文件
- **支持格式**：OpenSSH 格式、PEM 格式
- **优先级**：P0

#### F-005 终端模拟器
- **描述**：基于 xterm.js 的终端模拟器，支持完整的 VT100/xterm 转义序列
- **要求**：WebGL 渲染、自适应窗口大小、自定义字体和颜色
- **优先级**：P0

#### F-006 多标签页
- **描述**：支持多个 SSH 会话标签页，可切换、关闭、重命名
- **交互**：拖拽排序、快捷键切换、标签页状态指示
- **优先级**：P0

### 3.2 V0.5 — 实用版

#### F-007 SFTP 文件管理
- **描述**：集成 SFTP 文件浏览器
- **功能**：上传、下载、删除、重命名、编辑文本文件
- **界面**：双栏布局（本地 + 远程），支持拖拽传输
- **优先级**：P0

#### F-008 SSH 端口转发
- **描述**：支持本地转发、远程转发、动态转发（SOCKS 代理）
- **管理**：可视化转发规则管理，状态监控
- **优先级**：P0

#### F-009 配置加密导出/导入
- **描述**：将服务器列表、分组、配置导出为加密文件
- **格式**：JSON + AES-256-GCM 加密
- **用途**：跨设备迁移，配置备份
- **优先级**：P0

#### F-010 主题系统
- **描述**：内置亮色/暗色主题，支持自定义主题
- **自定义**：终端颜色方案、字体、UI 颜色
- **优先级**：P1

#### F-011 快捷键系统
- **描述**：全局快捷键和自定义快捷键
- **默认**：标签页切换、搜索、连接、断开等
- **优先级**：P1

### 3.3 V1.0 — AI 版（正式发布）

#### F-012 AI 危险命令拦截
- **描述**：识别高危命令（rm -rf、dd、mkfs 等），弹窗确认
- **实现**：本地规则 + AI 语义分析
- **优先级**：P0

#### F-013 AI 命令解释
- **描述**：选中命令后，AI 解释其功能和参数
- **交互**：右键菜单"解释命令"或快捷键触发
- **优先级**：P0

#### F-014 自然语言转命令
- **描述**：输入自然语言描述，AI 生成对应的 Shell 命令
- **交互**：专用输入框或快捷键激活
- **优先级**：P1

#### F-015 智能命令补全
- **描述**：基于服务器环境的上下文命令补全
- **优先级**：P1

#### F-016 多 AI 后端支持
- **描述**：支持 OpenAI / Claude / Ollama 本地模型
- **要求**：用户自带 API Key，不做中间代理
- **优先级**：P0

## 4. 非功能需求

### 4.1 性能
- 启动时间 < 2 秒
- 终端输入延迟 < 16ms（60fps）
- 内存占用 < 100MB（单会话）
- 支持同时打开 20+ 标签页

### 4.2 安全
- 所有密码/密钥 AES-256-GCM 加密存储
- 不以明文记录敏感信息到日志
- 导出文件加密保护
- AI 请求不传输服务器凭证

### 4.3 兼容性
- macOS 12+
- Windows 10+
- Ubuntu 20.04+ / Fedora 36+

### 4.4 国际化
- 初期支持中文、英文
- 架构支持 i18n 扩展

### 7.1 最新迭代说明

#### V0.12.0 — SSH ProxyJump 跳板机功能完整迭代

生产级运维场景核心需求：通过跳板机（Bastion Host）安全访问内网服务器。

- **单级 + 多级跳板机支持**（无限级数）
- **跳板机 Session 共享**（连接池，多个内网服务器复用同一跳板机）
- **SSH Agent Forwarding**（无需存储私钥）
- **Sidebar 特殊渲染**（徽章 + 引用计数 + 链路预览）
- **ConnectModal Tab 化重构**（授权信息 + SSH 隧道分离）

详见 [V0.12.0 完整迭代文档](./iterations/v0.12.0-proxyjump-bastion.md)

#### V0.13.0 — SFTP 面板完整增强迭代

SFTP 面板生产级增强：修复布局问题、完整右键菜单、剪贴板操作、权限管理。

- **Bug 修复**：文件列表滚动 + 状态栏遮挡
- **右键菜单系统**：主菜单 7 项 + 更多子菜单 7 项
- **剪贴板操作**：复制/剪切/粘贴/移动
- **权限修改**：chmod 操作，八进制权限编辑
- **文件信息查看**：模态对话框，显示详细属性

详见 [V0.13.0 完整迭代文档](./iterations/v0.13.0-sftp-enhancement-complete.md)

#### V0.14.0 — 终端字体管理完整迭代

终端字体体验全面升级：内置开源字体、智能选择器、自定义上传、实时热更新。

- **6 款内置开源字体**（JetBrains Mono / Fira Code / Cascadia Code / Source Code Pro / Hack / IBM Plex Mono）
- **智能字体选择器**（el-select 分组下拉，区分内置/自定义，字体预览渲染）
- **自定义字体上传**（.ttf/.otf/.woff/.woff2，存储于 ~/.termex/fonts/）
- **字体删除管理**（下拉列表内直接删除自定义字体）
- **终端实时热更新**（字体/字号更改即时生效，无需重启终端）

详见 [V0.14.0 完整迭代文档](./iterations/v0.14.0-font-management.md)

#### V0.15.0 — 终端搜索系统完整迭代

终端搜索体验全面建设：基于 8 款主流 SSH 客户端调研，实现三层搜索能力。

- **终端内文本搜索**（Cmd+F / Ctrl+F 浮动搜索栏，正则/大小写/全词匹配，匹配计数与导航）
- **持久化关键词高亮**（自定义规则自动着色 ERROR/WARNING 等关键词，Settings 管理，预设模板）
- **跨 Tab 搜索**（Cmd+Shift+F 搜索所有打开的终端 Tab，结果分组展示，点击跳转定位）

详见 [V0.15.0 完整迭代文档](./iterations/v0.15.0-terminal-search.md)

#### V0.16.0 — 自定义快捷键迭代

将硬编码快捷键改为用户可配置，Settings 面板中点击录入式绑定。

- **录入式快捷键绑定**（点击标签 → "Press shortcut..." → 按组合键即完成）
- **冲突检测**（自动检查是否与已有快捷键冲突，提示占用的 action 名称）
- **安全校验**（必须包含修饰键，系统保留快捷键如 Cmd+C/V 不可绑定）
- **重置能力**（单项重置 + 全局重置到默认值）
- **持久化**（仅存储差异项，与默认值合并加载，重启保持）

详见 [V0.16.0 完整迭代文档](./iterations/v0.16.0-custom-keybindings.md)

#### V0.17.0 — 服务器间 SFTP 文件传输迭代

对标 Termius 核心差异化功能：两台远程服务器之间直接传输文件，数据通过本地 32KB 流式中继。

- **双远程 SFTP 面板**（左右栏均可选择远程服务器或本地，模式选择下拉切换）
- **服务器间流式传输**（Server A → 32KB 中继 → Server B，恒定内存占用）
- **统一面板组件**（SftpFilePane 替代 LocalFilePane + RemoteFilePane，消除代码重复）
- **sftpStore 双栏重构**（单会话模型 → 双栏独立状态，保持向后兼容）
- **传输进度 + 错误处理**（实时进度条，断连错误提示，与现有传输共存）

详见 [V0.17.0 完整迭代文档](./iterations/v0.17.0-server-to-server-sftp.md)

#### V0.18.0 — SSH 代理连接协议支持

一次性补全 SSH 客户端常见代理协议，对标 PuTTY / SecureCRT / Termius 内置代理支持。

- **SOCKS5 / SOCKS4 / HTTP CONNECT 代理协议**（tokio-socks + 手工 HTTP CONNECT 实现）
- **独立代理表 + 引用模式**（Settings 全局管理代理列表，连接时选择共享代理）
- **ConnectModal 新增 Proxy Tab**（与 Authorization / SSH Tunnel 平级，选择预配置代理）
- **代理 + 跳板机组合**（Client → Proxy → Bastion → Target 完整链路支持）
- **代理密码安全存储**（OS Keychain + AES-256-GCM 加密，遵循现有凭证安全策略）

详见 [V0.18.0 完整迭代文档](./iterations/v0.18.0-proxy-protocols.md)
