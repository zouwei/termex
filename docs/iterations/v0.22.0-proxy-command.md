# V0.22.0 — ProxyCommand Support (Cloudflare Tunnel / Custom Transport)

## 背景

用户需要通过 Cloudflare Tunnel 访问内部 SSH 服务器。Cloudflare Tunnel 的 SSH 接入方式要求客户端执行 `cloudflared access ssh --hostname %h`，将子进程的 stdin/stdout 作为 SSH 传输通道 —— 这就是 OpenSSH 的 `ProxyCommand` 机制。

Termex v0.18.0 实现了 SOCKS5/SOCKS4/HTTP CONNECT/Tor 四种网络代理，但当时明确排除了 ProxyCommand（原因：GUI 应用执行任意 shell 命令的安全风险）。经评估，ProxyCommand 由用户在本地配置，与 OpenSSH `~/.ssh/config` 的信任模型一致，安全风险可控。

### 当前状态

- **已实现**: SOCKS5, SOCKS4, HTTP CONNECT (含 TLS/mTLS), Tor 代理
- **已实现**: `AsyncStream` trait 抽象 + `russh::client::connect_stream()` 自定义传输
- **已实现**: Proxy CRUD + 加密存储 + Keychain 集成
- **未实现**: ProxyCommand (子进程传输)

### 设计决策

- ProxyCommand 作为新的代理类型 `"command"` 加入现有代理体系，复用全部基础设施
- 命令通过 `sh -c` (Unix) / `cmd /C` (Windows) 执行，支持 shell 特性（管道、环境变量）
- 支持 OpenSSH 变量替换：`%h` (主机名), `%p` (端口), `%r` (用户名), `%%` (字面 %)
- 子进程在 SSH 断开时自动终止（Drop trait + kill_on_drop）
- 不需要新的 Cargo 依赖（`tokio::process` 已包含在 `tokio = { features = ["full"] }` 中）

## 一、核心实现：CommandStream

### 1.1 进程流包装器

新建 `src-tauri/src/ssh/proxy_command.rs`，实现将子进程 stdin/stdout 组合为双向异步流：

```rust
/// Bidirectional async stream wrapping a child process's stdin/stdout.
/// Implements AsyncRead (from stdout) + AsyncWrite (to stdin).
/// Child process is killed when stream is dropped.
pub struct CommandStream {
    stdout: ChildStdout,
    stdin: ChildStdin,
    child: Child,
}

impl AsyncRead for CommandStream {
    fn poll_read(...) -> Poll<io::Result<()>> {
        // delegate to self.stdout
    }
}

impl AsyncWrite for CommandStream {
    fn poll_write(...) -> Poll<io::Result<usize>> {
        // delegate to self.stdin
    }
    fn poll_flush(...) -> Poll<io::Result<()>> {
        // delegate to self.stdin
    }
    fn poll_shutdown(...) -> Poll<io::Result<()>> {
        // delegate to self.stdin
    }
}

impl Drop for CommandStream {
    fn drop(&mut self) {
        let _ = self.child.start_kill(); // non-blocking, tolerates "already exited"
    }
}
```

### 1.2 变量替换

```rust
/// Substitutes OpenSSH-style variables in a ProxyCommand string.
/// - %h → target hostname
/// - %p → target port
/// - %r → remote username
/// - %% → literal %
pub fn substitute_vars(command: &str, host: &str, port: u16, username: Option<&str>) -> String
```

### 1.3 连接入口

```rust
/// Spawns a ProxyCommand process and returns its stdin/stdout as AsyncStream.
/// Command is passed to `sh -c` (Unix) / `cmd /C` (Windows).
pub async fn connect_command(
    command: &str,
    target_host: &str,
    target_port: u16,
    username: Option<&str>,
) -> Result<Box<dyn AsyncStream>, SshError>
```

执行流程：
1. `substitute_vars()` 替换变量
2. `tokio::process::Command::new("sh").args(["-c", &cmd])` 启动子进程
3. `.stdin(Stdio::piped())`, `.stdout(Stdio::piped())`, `.stderr(Stdio::null())`
4. `.kill_on_drop(true)` 安全兜底
5. 提取 stdin/stdout 构造 `CommandStream`
6. 返回 `Box::new(stream)` → 传给 `russh::client::connect_stream()`

## 六、进程生命周期管理

```
连接:  用户点击连接
        → ProxyConfig { proxy_type: Command, command: "cloudflared ..." }
        → connect_command() 启动子进程, 返回 CommandStream
        → russh::client::connect_stream(stream) 建立 SSH
        → CommandStream 所有权转移给 russh

断开:  用户关闭 tab / 窗口关闭 / 网络断开
        → SshSession drop → russh handle drop → CommandStream drop
        → Drop::drop() 调用 child.start_kill()
        → 子进程终止（SIGKILL on Unix / TerminateProcess on Windows）
```

无需修改 `SshSession::disconnect()` 或窗口关闭清理逻辑 —— 现有的 drop 链路自动处理。

## 七、安全考量

| 风险 | 措施 |
|------|------|
| 任意命令执行 | 命令由用户在本地设置界面配置，信任模型与 OpenSSH `ProxyCommand` 一致 |
| 命令注入 | 不存在 —— 无远程输入影响命令字符串 |
| 子进程泄漏 | `Drop` trait + `kill_on_drop(true)` 双重保障 |
| 凭据泄漏 | 命令存储在 SQLCipher 加密数据库中 |
