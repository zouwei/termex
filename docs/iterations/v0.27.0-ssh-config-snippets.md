# v0.27.0 — SSH Config Import + Snippet Manager: SSH 配置导入与命令片段管理

> 状态：**计划中**
> 依赖：v0.26.0
> 关联：v0.22.0 (ProxyCommand)、v0.24.0 (Connection Chain)、v0.6.0 (AI NL2Cmd)

---

## 背景

### Feature 1: SSH Config Import

许多开发者在 `~/.ssh/config` 中配置了 50-200+ 台服务器。手动迁移到 Termex 是影响用户采纳的最大障碍。一键导入 = 即时价值。

| 功能 | 状态 |
|------|------|
| 手动创建 Server | 已实现 |
| `.termex` 格式导入导出 | 已实现 |
| ProxyJump / ProxyCommand / Connection Chain | 已实现 (v0.12/v0.22/v0.24) |
| **OpenSSH `~/.ssh/config` 解析导入** | **未实现** |
| **第三方格式导入 (Termius/CSV)** | **未实现** |

### Feature 2: Command Snippet Manager

高频用户每天重复输入相同命令。当前: 记住 -> 输入 -> 执行。目标: 保存一次 -> 搜索 -> 一键执行。Snippet Manager 是 AI NL2Cmd 的天然补充 — AI 生成的命令可直接保存为片段复用。

| 功能 | 状态 |
|------|------|
| AI NL2Cmd + Insert | 已实现 |
| AI Smart Autocomplete (v0.26.0) | 已实现 |
| **命令片段持久化/分组/搜索/变量模板** | **未实现** |
| **快速调用面板 (Command Palette 风格)** | **未实现** |

### 设计决策

1. SSH Config Parser 纯 Rust 手工解析，不依赖第三方 SSH config crate
2. `Include` 指令使用 `glob` crate 展开（新依赖）
3. 导入采用 Preview -> Confirm 两步流程
4. Snippet 存储在 SQLCipher 数据库（Migration V12）
5. 变量模板语法 `${VAR_NAME}`，执行时弹窗输入
6. 快速调用面板快捷键 `Cmd+Shift+S` / `Ctrl+Shift+S`

## 一、SSH Config Parser

**新建文件**: `src-tauri/src/ssh/config_parser.rs` (~350 行)

### 1.1 核心类型

```rust
/// A single parsed SSH config host entry.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct SshConfigEntry {
    pub host_alias: String,
    pub hostname: String,     // HostName (falls back to host_alias)
    pub port: u16,            // default 22
    pub user: String,         // default current user
    pub identity_file: Option<String>,   // IdentityFile (~ resolved)
    pub proxy_jump: Option<String>,      // ProxyJump
    pub proxy_command: Option<String>,   // ProxyCommand
    pub is_wildcard: bool,
    pub raw_options: HashMap<String, String>,
}

pub struct ParseResult {
    pub entries: Vec<SshConfigEntry>,
    pub errors: Vec<ParseError>,
}

pub struct ParseError { pub file: String, pub line: usize, pub message: String }
```

### 1.2 解析流程

```rust
pub fn parse_ssh_config(path: &Path) -> Result<ParseResult, String>
```

1. 逐行读取，跳过 `#` 注释和空行
2. 遇到 `Host` 指令 -> 开启新 block，支持多 pattern（空格分隔）
3. 遇到 `Include` 指令 -> `resolve_tilde()` + `glob::glob()` 展开 -> 递归解析（深度上限 10，循环引用保护）
4. 全局选项（`Host` 前的指令）隐式归入 `Host *` block
5. 解析完毕后 `merge_wildcards()` — 将 `Host *` / `Host *.example.com` 的默认值合并到具体 Host（first-match-wins，与 OpenSSH 语义一致）
6. 路径解析: `~` 展开使用 `dirs::home_dir()`；默认 User 使用 `whoami::username()`

### 1.3 通配符匹配

支持 `*`（任意字符串）和 `?`（单字符），以及 `!` 前缀排除。使用递归回溯匹配（SSH config 模式简单，无需 DP）。

### 2.1 Preview（预览，无副作用）

```rust
#[tauri::command]
pub fn ssh_config_preview(path: Option<String>) -> Result<PreviewResult, String>
```

- 默认路径 `~/.ssh/config`
- 返回所有解析出的 entry + parse errors，前端展示供用户勾选

### 2.2 Import（实际导入）

```rust
#[tauri::command]
pub fn ssh_config_import(
    state: State<'_, AppState>,
    path: Option<String>,
    selected_aliases: Vec<String>,  // 用户勾选的 host alias
) -> Result<ImportResult, String>
```

- `ImportResult { total, imported, skipped, errors: Vec<ImportError> }`
- 自动创建 "SSH Config Import" 分组，所有导入 server 归入
- 去重: 相同 `host + port + username` 视为已存在，跳过
- `auth_type = "key"` 当 IdentityFile 存在，否则 `"password"`
- `ProxyJump` -> 查找/创建 bastion server -> 写入 `connection_chain` (phase=pre)
- `ProxyCommand` -> 创建 Proxy 记录 (type=command) -> 写入 chain

### 9.3 变量执行流程

1. 点击 snippet -> 检测 `${VAR}` 变量
2. 有变量 -> 弹出 ElDialog，每个变量一个 ElInput
3. 用户填写 -> 点击 "Execute" -> `resolve_variables()` -> `ssh_write` 发送到活跃终端
4. 无变量 -> 直接发送
5. 更新 `usage_count` + `last_used_at`

### 9.4 SnippetPalette

类似 VS Code Command Palette 的浮动面板:

- 全局快捷键 `Cmd+Shift+S` / `Ctrl+Shift+S` 打开
- 输入框 + 搜索结果列表（150ms debounce）
- 上下键选择 + Enter 执行 + Escape 关闭
- 结果按 usage_count 排序（常用优先）
- Teleport to body，fixed 定位，20vh 距顶

## 十五、已知限制

1. **PuTTY Registry Import**: Windows only (需 `winreg` crate)，v0.27.0 仅预留接口不实现
2. **SSH Config `Match` 指令**: 条件匹配复杂度高，v0.27.0 跳过 `Match` 块，仅解析 `Host` 块。遇到 `Match` 指令时在 `ParseResult.errors` 中记录 warning（`"Match directive skipped at line {n}, only Host blocks are supported"`），前端 Preview 展示该 warning
3. **Snippet 跨设备同步**: 仅本地存储，需等待 v0.30.0 (Team Collaboration) 支持共享片段
4. **Snippet 导出**: `.termex` 格式尚未包含 snippet 数据。本迭代确保 config_import 对 snippets 字段做向前兼容处理（缺失时跳过，不报错）。完整 snippet 导出/导入功能随 v0.30.0 团队协作一并支持
5. **变量默认值**: 不支持 `${VAR:-default}` 语法，后续版本扩展
6. **导入凭证处理**: SSH Config 中的 IdentityFile 仅记录路径，passphrase 不在导入时采集。首次连接时若密钥需要 passphrase，走标准的 Termex 密码输入流程（与手动创建服务器一致），输入后存入 OS Keychain
