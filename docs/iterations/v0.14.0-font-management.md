# Termex v0.14.0 — 终端字体管理完整迭代

## 迭代概述

v0.14.0 解决终端字体选择受限的问题，提供完整的字体管理体验：

- **Bug Fix**: Font Family 从纯文本输入改为智能下拉选择器
- **Feature**: 内置 6 款开源免费等宽字体（.woff2 打包）
- **Feature**: 用户自定义字体上传、选择、删除
- **Feature**: 终端字体/字号实时热更新（无需重新打开终端）
- **Bug Fix**: useTerminal 硬编码字体改为读取 settingsStore

---

## 背景与痛点

### 问题 1: 字体选择受限

**现象**: Font Family 设置项是纯文本输入框，用户必须手动输入字体名称，且仅预设 JetBrains Mono。

**用户诉求**: 希望能选择 MesloLGS NF 等其他字体，需要下拉列表方便选择。

### 问题 2: 内置字体许可不明确

**现象**: 当前依赖系统安装的字体，未内置任何字体文件。如果用户系统未安装 JetBrains Mono，终端将回退到系统默认等宽字体。

**用户诉求**: 确保内置字体全部开源免费，开箱即用。

### 问题 3: 无法使用自定义字体

**现象**: 用户无法上传自己的字体文件到 Termex 中使用。

**用户诉求**: 允许上传 .ttf/.otf/.woff/.woff2 字体文件，上传后自动出现在 Font Family 选择列表中。

### 问题 4: 字体更改不实时生效

**现象**: `useTerminal.ts` 硬编码了字体配置 `fontFamily: "'JetBrains Mono', 'Fira Code', 'Cascadia Code', monospace"`，不读取 settingsStore。更改字体设置后需重新打开终端才能生效。

## 核心设计

### 1. UI 布局重构

#### 当前布局

```
┌─────────────────────┬──────────────────────┐
│ Font Family          │ Font Size            │
│ [text input]         │ [number input]       │
└─────────────────────┴──────────────────────┘
```

#### 新布局

```
┌─────────────────────┬──────────────────────┐
│ Font Family          │ Upload Font          │
│ [el-select ▼]       │ [Browse...] button   │
├─────────────────────┴──────────────────────┤
│ Font Size                                   │
│ [number input]                              │
└─────────────────────────────────────────────┘
```

- Font Family：`el-select` 下拉选择器（filterable），替代纯文本输入
- Font Size：从第一行右侧移至第二行
- Upload Font：占据原 Font Size 位置，提供字体上传入口

### 2. Font Family 下拉菜单结构

```
el-select (filterable)
├── [Group: Built-in Fonts / 内置字体]
│   ├── JetBrains Mono        ← 用该字体自身渲染预览
│   ├── Fira Code
│   ├── Cascadia Code
│   ├── Source Code Pro
│   ├── Hack
│   └── IBM Plex Mono
└── [Group: Custom Fonts / 自定义字体]   ← 仅有上传字体时显示
    ├── MesloLGS NF           [✕]  ← hover 显示删除按钮
    └── My Custom Font        [✕]
```

**交互细节**：
- 每个选项用该字体自身渲染 → `<span :style="{ fontFamily: name }">{{ name }}</span>`
- 自定义字体右侧有删除图标 `@click.stop` 阻止选中
- 删除前弹出 `ElMessageBox.confirm` 确认对话框

- 通过 CSS `@font-face` 声明加载
- 预计增加约 200-400KB 应用体积

### 4. 自定义字体管理

#### 上传流程

```
用户点击 Browse... → 选择 .ttf/.otf/.woff/.woff2 文件
    ↓
前端 FileReader 读取 ArrayBuffer
    ↓
调用 fonts_upload(fileName, data) → 后端保存到 ~/.termex/fonts/
    ↓
前端 FontFace API 动态注册字体
    ↓
字体出现在 Font Family 下拉列表 "Custom Fonts" 分组
```

#### 删除流程

```
下拉列表中点击自定义字体的 ✕ 图标
    ↓
ElMessageBox.confirm 确认对话框
    ↓
调用 fonts_delete(fileName) → 后端删除 ~/.termex/fonts/ 中的文件
    ↓
前端 document.fonts.delete() 卸载字体
    ↓
若被删字体是当前选中字体 → 自动回退到 JetBrains Mono
```

#### 持久化策略

- 字体文件持久在 `~/.termex/fonts/` 目录
- 应用启动时 `loadCustomFonts()` 扫描目录并通过 FontFace API 重新注册
- `fontFamily` 设置值（仅字体名）持久在 settings 数据库
- 字体文件手动删除时优雅降级（CSS 回退到 monospace）

### 5. 后端命令设计

#### 新增模块：`src-tauri/src/commands/fonts.rs`

##### 数据结构

```rust
#[derive(Debug, Clone, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct CustomFont {
    pub name: String,       // 显示名（文件名去扩展名，- _ 替换为空格）
    pub file_name: String,  // 原始文件名
    pub path: String,       // 完整路径
    pub size: u64,          // 文件大小（字节）
}
```

##### 命令清单

| 命令 | 签名 | 功能 |
|------|------|------|
| `fonts_list_custom` | `() -> Vec<CustomFont>` | 扫描 `~/.termex/fonts/`，过滤 .ttf/.otf/.woff/.woff2 |
| `fonts_upload` | `(file_name: String, data: Vec<u8>) -> CustomFont` | 写入字体文件，返回元数据 |
| `fonts_delete` | `(file_name: String) -> ()` | 删除指定字体文件 |
| `fonts_read` | `(file_name: String) -> Vec<u8>` | 读取字体文件字节（供前端 FontFace API 加载） |

**安全措施**：
- `file_name` 参数取 basename，拒绝含 `..` 或 `/` 的路径（防路径穿越）
- 仅允许 `.ttf`、`.otf`、`.woff`、`.woff2` 扩展名

##### 辅助函数

```rust
fn get_fonts_dir() -> Result<PathBuf, String>
```

复用 `get_app_data_dir()` 模式（同 `local_ai.rs`），返回 `~/.termex/fonts/`。

### 6. 前端字体加载工具

#### 新增：`src/utils/fontLoader.ts`

| 函数 | 功能 |
|------|------|
| `loadCustomFont(font)` | 通过 Tauri 命令读取字节 → Blob URL → FontFace API 注册 |
| `unloadCustomFont(name)` | 从 `document.fonts` 移除指定字体 |
| `loadAllCustomFonts(fonts[])` | 批量加载所有自定义字体 |
| `buildFontFamilyCSS(name)` | 构建 `"'FontName', monospace"` CSS 字符串 |

**加载策略**：

采用 Blob URL 方式避免 Tauri v2 CSP 限制：
```typescript
const data = await tauriInvoke<number[]>('fonts_read', { fileName });
const blob = new Blob([new Uint8Array(data)], { type: 'font/woff2' });
const url = URL.createObjectURL(blob);
const face = new FontFace(name, `url(${url})`);
await face.load();
document.fonts.add(face);
```

### 7. Store 扩展

#### 修改：`src/stores/settingsStore.ts`

**状态变更**：
- `fontFamily` 默认值：`"'JetBrains Mono', 'Fira Code', monospace"` → `"JetBrains Mono"`（仅字体名）
- 新增 `customFonts: ref<CustomFont[]>([])`

**新增 Actions**：

| Action | 功能 |
|--------|------|
| `loadCustomFonts()` | 调用 `fonts_list_custom` + `loadAllCustomFonts()` |
| `uploadFont(fileName, data)` | 调用 `fonts_upload` + 注册 FontFace + 更新列表 |
| `deleteFont(fileName)` | 调用 `fonts_delete` + 卸载 FontFace + 更新列表 + 必要时回退默认字体 |

**迁移处理**：
```typescript
// loadAll() 中处理旧格式 "'JetBrains Mono', 'Fira Code', monospace" → "JetBrains Mono"
case "fontFamily":
  const match = value.match(/^'?([^',]+)'?/);
  fontFamily.value = match ? match[1].trim() : DEFAULT_FONT;
  break;
```

## 设计决策

| 决策 | 方案 | 原因 |
|------|------|------|
| 字体打包格式 | .woff2 | 最高压缩率，所有现代浏览器/WebView 支持 |
| 仅打包 Regular 权重 | 省空间 | xterm.js 自行合成粗体，无需额外权重文件 |
| 自定义字体加载方式 | Blob URL + FontFace API | 避免 Tauri v2 CSP 限制 file:// 协议 |
| fontFamily 存储格式 | 纯字体名 | CSS 回退栈在使用时构建，存储更简洁 |
| 字体存储位置 | ~/.termex/fonts/ | 复用现有 ~/.termex/ 目录结构 |
| 字体名提取方式 | 文件名去扩展名 + 替换分隔符 | 简单可靠，无需引入字体解析库 |
