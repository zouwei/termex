# V0.23.0 — Portable Mode (USB 便携运行)

## 背景

用户需要将 Termex 放在 U 盘上随身携带，插入任意电脑即可使用，无需安装，配置和数据跟随 U 盘迁移。这是运维工程师和数字游民的核心场景 —— 在公司、客户现场、网吧等不同环境中使用同一套服务器配置和凭据。

### 当前状态

Termex 有 **6 处数据写入系统用户目录**，无法便携：

| 数据 | 路径来源 | 存储位置（macOS/Linux） |
|------|---------|----------------------|
| 数据库 | `dirs::data_dir()` | `~/.local/share/termex/termex.db` |
| 凭据 | OS Keychain | macOS Keychain / GNOME Keyring |
| 字体 | 硬编码 `~/.termex/` | `~/.termex/fonts/` |
| AI 模型 | 硬编码 `~/.termex/` | `~/.termex/models/` |
| 录屏 | `dirs::data_dir()` | `~/.local/share/termex/recordings/` |
| 二进制 | 硬编码 `~/.termex/` | `~/.termex/bin/` |

### 设计决策

- 便携模式通过 **标记文件** `.portable` 检测（放在可执行文件同目录下即启用）
- 便携模式下所有数据存储在可执行文件旁的 `data/` 目录，不写入系统目录
- 凭据强制使用 **AES-256-GCM + Master Password** 路径（不使用 OS Keychain）
- 便携模式下禁用本地 AI 模型下载（模型文件 1-8GB，不适合 U 盘）
- 普通安装模式行为不变（零影响）
- 分发方式：zip 解压即用，不走安装包

## 一、核心模块：统一路径解析

### 1.1 `paths.rs` — 全局路径解析器

新建 `src-tauri/src/paths.rs`，所有模块通过此模块获取数据路径：

```rust
use std::path::PathBuf;
use std::sync::OnceLock;

/// Portable mode state, initialized once at app startup.
static PORTABLE_ROOT: OnceLock<Option<PathBuf>> = OnceLock::new();

/// Initializes path resolver. Call once at app startup.
/// Checks for `.portable` file next to the executable.
pub fn init() {
    PORTABLE_ROOT.get_or_init(|| {
        let exe = std::env::current_exe().ok()?;
        let exe_dir = exe.parent()?;
        if exe_dir.join(".portable").exists() {
            Some(exe_dir.join("data"))
        } else {
            None
        }
    });
}

/// Returns true if running in portable mode.
pub fn is_portable() -> bool {
    PORTABLE_ROOT.get().map(|r| r.is_some()).unwrap_or(false)
}

/// Data directory: `<exe>/data/` (portable) or `~/.local/share/termex/` (installed).
pub fn data_dir() -> PathBuf {
    if let Some(Some(root)) = PORTABLE_ROOT.get() {
        return root.clone();
    }
    dirs::data_dir()
        .unwrap_or_else(|| PathBuf::from("."))
        .join("termex")
}

/// App directory: `<exe>/data/` (portable) or `~/.termex/` (installed).
pub fn app_dir() -> PathBuf {
    if let Some(Some(root)) = PORTABLE_ROOT.get() {
        return root.clone();
    }
    dirs::home_dir()
        .unwrap_or_else(|| PathBuf::from("."))
        .join(".termex")
}

/// Database file path.
pub fn db_path() -> PathBuf {
    data_dir().join("termex.db")
}

/// Custom fonts directory.
pub fn fonts_dir() -> PathBuf {
    app_dir().join("fonts")
}

/// Session recordings directory.
pub fn recordings_dir() -> PathBuf {
    data_dir().join("recordings")
}

/// AI models directory.
pub fn models_dir() -> PathBuf {
    app_dir().join("models")
}

/// AI binary directory (llama-server etc.).
pub fn bin_dir() -> PathBuf {
    app_dir().join("bin")
}
```

### 1.2 各模块改用 `paths::`

替换所有硬编码路径：

| 模块 | 当前代码 | 改为 |
|------|---------|------|
| `storage/db.rs` | `dirs::data_dir().join("termex/termex.db")` | `paths::db_path()` |
| `commands/fonts.rs` | `dirs::home_dir().join(".termex/fonts")` | `paths::fonts_dir()` |
| `recording/recorder.rs` | `dirs::data_dir().join("termex/recordings")` | `paths::recordings_dir()` |
| `commands/local_ai.rs` | `dirs::home_dir().join(".termex/models")` | `paths::models_dir()` |
| `commands/local_ai.rs` | `dirs::home_dir().join(".termex/bin")` | `paths::bin_dir()` |

## 二、凭据存储降级

### 2.1 便携模式下强制 Master Password

便携模式无法依赖 OS Keychain（不同电脑的 keychain 不共享）。

```rust
// keychain/mod.rs — init() 函数修改

pub fn init() {
    if paths::is_portable() {
        // 便携模式: 跳过 keychain 初始化，标记为不可用
        // 凭据将使用 AES-256-GCM + Master Password 加密后存入 DB
        return;
    }
    // 正常模式: 现有 keychain 初始化逻辑
    // ...
}

pub fn is_available() -> bool {
    if paths::is_portable() {
        return false; // 强制走 AES fallback
    }
    // 现有检测逻辑
}
```

现有 fallback 路径（AES-256-GCM + Argon2id KDF）已完整实现，便携模式直接复用。

### 2.2 首次启动 Master Password 弹窗

便携模式首次启动时，前端检测 `is_portable` → 强制弹出 Master Password 设置对话框。

后续启动时弹出解锁对话框（输入 Master Password 解密 DB 中的凭据）。

## 三、便携模式下的功能限制

| 功能 | 便携模式行为 |
|------|------------|
| SSH 连接 | 正常 |
| SFTP | 正常 |
| 端口转发 | 正常 |
| tmux 持久会话 | 正常 |
| Git Auto Sync | 正常 |
| AI 助手（远程 API） | 正常（API key 存 DB，AES 加密） |
| **本地 AI 模型** | **禁用**（models/ 目录不创建，下载入口隐藏） |
| **OS Keychain** | **不使用**（强制 Master Password） |
| 自定义字体 | 正常（fonts/ 跟随 data/ 目录） |
| 配置导出/导入 | 正常 |
| 录屏 | 正常（recordings/ 跟随 data/ 目录） |

## 五、分发方式

### 5.1 构建脚本

GitHub Actions 新增 portable 构建步骤：

```yaml
- name: Package portable (zip)
  run: |
    mkdir -p portable
    cp target/release/termex portable/       # 可执行文件
    touch portable/.portable                  # 标记文件
    mkdir -p portable/data                    # 数据目录（空）
    zip -r termex-portable-${{ matrix.os }}.zip portable/
```

### 5.2 产物

| 平台 | 安装包（现有） | 便携包（新增） |
|------|-------------|-------------|
| macOS ARM | `Termex_aarch64.dmg` | `termex-portable-macos-arm64.zip` |
| macOS Intel | `Termex_x64.dmg` | `termex-portable-macos-x64.zip` |
| Windows | `Termex_x64.msi` | `termex-portable-windows-x64.zip` |
| Linux | `.deb` / `.rpm` / `.AppImage` | `termex-portable-linux-x64.zip` |

### 5.3 U 盘目录结构

```
USB/
├── Termex.exe          (或 termex / Termex.app)
├── .portable           (标记文件，0 字节)
└── data/
    ├── termex.db       (SQLCipher 加密数据库)
    ├── fonts/          (自定义字体)
    └── recordings/     (录屏文件)
```

## 六、安全考量

| 风险 | 措施 |
|------|------|
| U 盘丢失 → 数据泄露 | SQLCipher 加密 + AES-256-GCM 凭据加密，无 Master Password 无法解密 |
| 暴力破解 Master Password | Argon2id KDF（m=64MB, t=3, p=4），每次尝试约 0.5 秒 |
| 恶意主机篡改可执行文件 | 用户自行验证（超出应用范围，建议使用加密 U 盘） |
| 临时文件泄露 | 便携模式不写入系统目录，`/tmp` 中的临时文件随系统清理 |
