> 状态：**计划中**
> 依赖：v0.32.0
> 关联：面向 K8s/容器用户群体，SSH 客户端进化为云原生运维工具

## 背景

### 市场趋势

2026 年，大量开发者的日常运维工作从传统 SSH 登录服务器转变为 `kubectl exec` 进入容器。Kubernetes 已成为生产环境的事实标准，但现有 SSH 客户端（包括 Termius、Tabby、WindTerm）都没有原生集成 K8s 能力。

专用的 K8s 管理工具（如 Lens、k9s）虽然存在，但用户需要在 SSH 客户端和 K8s 工具之间频繁切换。Termex 如果能在同一个界面内同时管理传统 SSH 服务器和 K8s Pod，将形成独家竞争力。

### 核心目标

1. **kubectl exec 集成** — 在 Termex 中直接 exec 进入 Pod/Container
2. **AWS SSM Session Manager** — 无需 SSH 端口，通过 SSM 连接 EC2 实例
3. **K8s 资源浏览** — 浏览 Cluster → Namespace → Pod，一键连接
4. **容器日志流** — 实时 tail 容器日志

### 竞品调研

| 产品 | K8s exec | AWS SSM | Pod 浏览 | 日志流 |
|------|----------|---------|---------|--------|
| Lens | ✅ | ❌ | ✅ | ✅ |
| k9s | ✅ | ❌ | ✅ | ✅ |
| Termius | ❌ | ❌ | ❌ | ❌ |
| Tabby | ❌ | ❌ | ❌ | ❌ |
| **Termex** | **✅** | **✅** | **✅** | **✅** |

## 一、K8s Exec 集成

### 1.1 架构设计

Termex 不直接实现 K8s API — 而是调用用户本地的 `kubectl` 命令行工具。这保持了 Termex "纯本地应用"的定位，无需内嵌 K8s SDK。

```
用户 → Termex → 本地 kubectl → K8s API Server → Pod
```

**前提条件**：
- 用户本地已安装 `kubectl`
- 已配置 `~/.kube/config`（或环境变量 `KUBECONFIG`）
- 对目标集群有 `exec` 权限

### 1.3 K8s Exec Session

通过本地 PTY 执行 `kubectl exec` 实现终端连接：

**新文件**: `src-tauri/src/cloud/kube_exec.rs`

```rust
use portable_pty::{CommandBuilder, NativePtySystem, PtySize, PtySystem};

/// 通过 kubectl exec 打开 Pod 内的 shell
pub fn kube_exec_open(
    context: &str,
    namespace: &str,
    pod: &str,
    container: Option<&str>,
    shell: Option<&str>,  // 默认 /bin/sh
    cols: u16,
    rows: u16,
) -> Result<PtySession, String> {
    let pty_system = NativePtySystem::default();
    let pair = pty_system.openpty(PtySize { rows, cols, pixel_width: 0, pixel_height: 0 })
        .map_err(|e| e.to_string())?;

    let mut cmd = CommandBuilder::new("kubectl");
    cmd.args(["--context", context, "-n", namespace, "exec", "-it", pod]);
    
    if let Some(c) = container {
        cmd.args(["-c", c]);
    }
    
    cmd.args(["--", shell.unwrap_or("/bin/sh")]);
    cmd.env("TERM", "xterm-256color");

    let child = pair.slave.spawn_command(cmd).map_err(|e| e.to_string())?;
    // ... 与 local_pty 相同的读写流管理
}
```

## 二、AWS SSM Session Manager

### 2.1 架构

```
用户 → Termex → 本地 aws ssm → AWS API → EC2 Instance
```

**前提条件**：
- AWS CLI v2 已安装
- SSM Session Manager Plugin 已安装
- AWS credentials 已配置（~/.aws/credentials 或环境变量）
- 目标 EC2 有 SSM Agent 运行

### 2.2 实现

同样通过本地 PTY 执行 `aws ssm start-session`：

```rust
/// 通过 AWS SSM 连接 EC2 实例
pub fn ssm_session_open(
    instance_id: &str,
    profile: Option<&str>,
    region: Option<&str>,
    cols: u16,
    rows: u16,
) -> Result<PtySession, String> {
    let mut cmd = CommandBuilder::new("aws");
    cmd.args(["ssm", "start-session", "--target", instance_id]);
    
    if let Some(p) = profile {
        cmd.args(["--profile", p]);
    }
    if let Some(r) = region {
        cmd.args(["--region", r]);
    }
    
    cmd.env("TERM", "xterm-256color");
    // ... PTY spawn
}
```

### 3.3 连接流程

**K8s exec**:
1. 用户在 Cloud 面板右键 Pod → "Connect"
2. 弹出 KubeConnectDialog：选择 container（多容器 Pod）、shell（/bin/sh, /bin/bash, /bin/zsh）
3. 确认 → 新建 Tab → 通过 local PTY 执行 `kubectl exec -it`
4. Tab 标题显示: `pod-name [k8s]`

**AWS SSM**:
1. 用户在 Cloud 面板右键实例 → "Connect"
2. 确认 → 新建 Tab → 通过 local PTY 执行 `aws ssm start-session`
3. Tab 标题显示: `instance-name [ssm]`

**日志流**:
1. 右键 Pod → "View Logs"
2. 弹出 KubeLogViewer（只读 xterm.js）
3. 实时显示 `kubectl logs -f pod/xxx` 输出
4. 可选: 选择 container、--since、--tail 参数

## 四、Session 类型扩展

当前 Termex 的 session 类型只有 `ssh` 和 `local`。需要扩展：

```typescript
export type SessionType = 'ssh' | 'local' | 'kube-exec' | 'ssm' | 'kube-logs';
```

在 `sessionStore` 中，根据 `sessionType` 决定：
- Tab 图标（SSH: 终端图标，K8s: K8s logo，SSM: AWS 图标）
- 状态栏信息（SSH 显示主机名，K8s 显示 pod/container，SSM 显示 instance-id）
- SFTP 可用性（只有 SSH 类型支持 SFTP）
- 监控可用性（SSH 和 SSM 支持监控，K8s exec 不支持）

## 七、安全考虑

1. **凭证管理**：Termex 不存储任何云凭证 — 依赖用户本地的 `~/.kube/config` 和 `~/.aws/credentials`
2. **命令注入防护**：所有用户输入（pod 名、instance-id）在传给 `Command::new()` 时作为 args 而非 shell 字符串，避免注入
3. **权限最小化**：只需要 `pods/exec`、`pods/log`、`ssm:StartSession` 权限
4. **审计日志**：K8s exec 和 SSM 连接记录到 audit_logs 表

## 十一、风险与缓解

| 风险 | 严重性 | 缓解措施 |
|------|--------|---------|
| kubectl 未安装 | 低 | 引导页面 + 安装提示 |
| K8s API 响应慢 | 中 | 异步加载 + loading 状态 + 超时设置 |
| RBAC 权限不足 | 中 | 清晰的错误提示 + 需要的权限列表 |
| SSM Plugin 版本不兼容 | 低 | 版本检测 + 最低版本要求 |
| Pod exec 权限被禁 | 低 | PodSecurityPolicy 检测 + 提示 |
