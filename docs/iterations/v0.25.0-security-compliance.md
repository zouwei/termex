# V0.25.0 — Security Compliance (GDPR / ISO 27001 / GB/T 22239)

## Summary

Comprehensive security hardening across the entire stack, targeting three compliance frameworks:
- **GDPR** (EU General Data Protection Regulation)
- **ISO 27001:2022** (Information Security Management)
- **GB/T 22239-2019** (China Classified Protection of Information Security, Level 3)

## Changes

### P0: SSH Host Key TOFU Verification (Critical)

**Files**: `ssh/host_key.rs` (new), `ssh/auth.rs`, `ssh/session.rs`, `commands/ssh.rs`, `state.rs`

- Implemented Trust On First Use (TOFU) model matching OpenSSH behavior
- `check_server_key()` now captures the server's public key via shared `Arc<Mutex<Option<PublicKey>>>`
- Post-connect verification against the `known_hosts` database table (V1 schema, previously unused)
- Three verification outcomes: `Trusted`, `NewHost` (auto-trust silently), `KeyChanged` (MITM warning)
- On key change: emits `ssh://host-key-changed` event to frontend, waits up to 60s for user decision
- Frontend `HostKeyDialog.vue` shows fingerprint comparison with red MITM warning
- New IPC commands: `ssh_host_key_respond`, `ssh_known_hosts_list`, `ssh_known_hosts_remove`
- SHA-256 fingerprints stored in standard `SHA256:base64` format

### P0: Content Security Policy

**File**: `tauri.conf.json`

- Enabled CSP: `default-src 'self'; script-src 'self' 'wasm-unsafe-eval'; style-src 'self' 'unsafe-inline'; font-src 'self' data:; connect-src 'self' https:; img-src 'self' data: blob:`
- Blocks XSS injection, unauthorized script execution, and non-HTTPS resource loading
- `wasm-unsafe-eval` required for xterm.js WebGL renderer
- `unsafe-inline` for styles required by Element Plus

### P1: Memory Credential Security (zeroize)

**Files**: `Cargo.toml`, `crypto/kdf.rs`, `crypto/aes.rs`, `state.rs`, `commands/crypto.rs`

- Added `zeroize = { version = "1", features = ["zeroize_derive"] }` dependency
- `derive_key()` returns `Zeroizing<[u8; 32]>` — key is automatically zeroed on drop
- `AppState.master_key` is now `RwLock<Option<Zeroizing<[u8; 32]>>>`
- `master_password_lock()` no longer needs manual `fill(0)` — just sets `None`
- AES decrypt intermediate buffer wrapped in `Zeroizing<Vec<u8>>`

### P1: Audit Logging System

**Files**: `audit/mod.rs` (new), `commands/audit.rs` (new), `storage/migrations.rs` (V11)

- New `audit_log` table with indexes on `timestamp` and `event_type`
- 16 event types: master password ops, SSH connect/disconnect, host key trust, server CRUD, config export/import, credential access
- `audit::log()` — fire-and-forget logging (never blocks the operation)
- `audit::cleanup()` — 90-day retention, runs on startup
- IPC commands: `audit_log_list` (paginated, filterable), `audit_log_cleanup`
- Audit calls inserted in: `crypto.rs` (password ops), `ssh.rs` (connect/disconnect)

### P1: Session Idle Timeout

**File**: `src/composables/useIdleTimer.ts` (new)

- Monitors `mousemove`, `keydown`, `mousedown`, `touchstart`, `scroll` events
- Configurable timeout (default 30 min, 0 = disabled)
- Returns `{ isIdle, resume }` reactive refs
- Ready for integration in App.vue with lock screen overlay

### P2: Password Strength Policy

**Files**: `crypto/password_policy.rs` (new), `commands/crypto.rs`

- Minimum 8 characters, must contain uppercase + lowercase + digit
- Common password blacklist (19 entries)
- Context check: rejects passwords containing "termex"
- Score 0-4 (weak/fair/good/strong) with user-facing feedback messages
- `master_password_set` and `master_password_change` enforce minimum score >= 2
- New IPC command: `check_password_strength` for real-time UI feedback

### P2: Authentication Failure Lockout

**File**: `commands/crypto.rs`

- In-memory failure counter (resets on app restart — by design, not persistent)
- 5 failures: 30-second lockout
- 10 failures: 5-minute lockout
- Successful verification resets counter
- Returns remaining lockout seconds in error message

### P2: GDPR Data Erasure

**File**: `commands/privacy.rs` (new)

- `privacy_erase_all_data`: one-click deletion of all user data
  - Clears all database tables (servers, keys, providers, settings, known_hosts, audit_log)
  - Resets OS keychain to empty `{}`
  - Deletes recording files, custom fonts, local AI models
- `privacy_data_summary`: counts stored data items for privacy dashboard
- Compliant with GDPR Art.17 (Right to Erasure) and Art.20 (Data Portability)

### P3: SSH Strong Algorithm Configuration

**File**: `ssh/session.rs`

- Configured `russh::Preferred` to exclude weak MAC algorithms
- Removed: `hmac-sha1-etm`, `hmac-sha1`
- Kept: `hmac-sha2-512-etm`, `hmac-sha2-256-etm`, `hmac-sha2-512`, `hmac-sha2-256`
- Key exchange, cipher, and compression use russh defaults (already strong: chacha20-poly1305, aes256-gcm, curve25519)
