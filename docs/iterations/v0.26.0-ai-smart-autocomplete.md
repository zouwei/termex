## 背景

### v0.6.0 现状评估

v0.6.0 规划了 NL2Cmd + Smart Autocomplete 两大功能。**NL2Cmd 已完整实现**，但 Smart Autocomplete 部分（终端内联补全）**完全缺失**：

| 功能 | 状态 | 说明 |
|------|------|------|
| AI 侧边面板 (AiPanel) | 已完成 | Ctrl+Shift+I 切换，NL 描述 -> 命令 |
| NL2Cmd 后端 | 已完成 | `ai_nl2cmd` 命令，流式响应 |
| 命令解释 | 已完成 | `ai_explain_command`，流式解释 |
| 危险命令检测 | 已完成 | `ai_check_danger`，40+ 正则规则 |
| 多 Provider 支持 | 已完成 | 12 种 Provider (Claude/OpenAI/Gemini/Ollama/Local 等) |
| 本地 AI 引擎 | 已完成 | llama-server 管理，模型下载/启停 |
| **终端内联补全** | **未实现** | 无 ghost text、无补全弹窗、无实时感知 |
| **快速命令选择 UI** | **未实现** | 无终端内嵌弹出面板 |
| **本地 AI 默认优先** | **未实现** | 需手动配置，未自动使用本地模型 |

### 核心痛点

1. **AI 补全与终端脱节** — 用户必须切换到侧边面板手动输入自然语言描述，无法在终端内实时获得建议
2. **无内联 ghost text** — 缺少类似 GitHub Copilot / Fig 的打字时智能提示体验
3. **命令自动执行** — 当前 "Insert" 按钮会追加 `\n` 自动执行命令，用户无法先审查再确认
4. **本地模型未优先** — 即使已下载并启动本地模型，仍需手动设为默认

### 目标

实现 IDE 风格的终端内联智能补全：

- 用户在终端输入命令时，**自动显示 ghost text 建议**（类似代码补全）
- 提供**快速选择弹窗**展示多个候选命令
- **默认使用本地 AI 模型**，零配置即可使用
- 补全只插入命令文本，**不自动执行**（用户自行按 Enter 确认）

## 一、命令行状态追踪

### 1.1 问题

终端输入通过 [useTerminal.ts:131-138](src/composables/useTerminal.ts#L131-L138) 直接发送到远程 SSH 服务器，前端无法感知用户当前正在输入什么命令。要实现实时补全，必须追踪命令行状态。

### 1.2 设计方案

采用**终端缓冲区解析**作为主要机制 — 读取 xterm.js `terminal.buffer.active` 中光标所在行的文本，提取出 shell 提示符之后的命令部分。

**为什么不用本地按键缓冲？**
- 远程 shell 的 tab 补全、`!!` 展开、Ctrl+R 搜索等会修改命令行内容
- xterm.js buffer 反映的是 shell 实际回显的结果，是 ground truth
- 本地按键追踪会与真实状态产生漂移

### 1.3 实现

**新建文件**: `src/types/commandTracker.ts`

```typescript
/** 命令行追踪状态 */
export interface CommandLineState {
  /** 当前输入的命令文本（已剥离 prompt） */
  command: string;
  /** 光标在命令中的位置 (0-based) */
  cursorPos: number;
  /** 是否处于 shell 提示符（可以输入命令） */
  atPrompt: boolean;
  /** 最后更新时间戳（用于 debounce） */
  lastUpdated: number;
}

/** 终端模式 */
export type TerminalMode =
  | "shell"       // 在 shell 提示符，用户可以输入命令
  | "running"     // 命令正在执行（输出流中）
  | "alternate"   // 备用屏幕缓冲区（vim/less/htop 等全屏程序）
  | "unknown";    // 初始状态
```

**新建文件**: `src/composables/useCommandTracker.ts` (~200 行)

遵循 [useKeywordHighlight.ts](src/composables/useKeywordHighlight.ts) 的 composable 模式：接受 `getTerminal` 回调，挂载 `onWriteParsed` 监听。

核心逻辑：

```typescript
export function useCommandTracker(
  getTerminal: () => Terminal | null,
  sessionId: Ref<string>,
) {
  const state = ref<CommandLineState>({ command: "", cursorPos: 0, atPrompt: false, lastUpdated: 0 });
  const terminalMode = ref<TerminalMode>("unknown");

  // 1. terminal.onData — 追踪用户输入时机（仅用于 debounce 触发）
  //    Enter (\r) → atPrompt=false, terminalMode='running'
  //    Ctrl+C (\x03) → 重置命令

  // 2. terminal.onWriteParsed — 远程回显后重新从 buffer 提取命令
  //    读取 buffer.active.getLine(cursorY).translateToString(true)
  //    用 prompt 启发式匹配提取命令部分

  // 3. 备用屏幕检测
  //    buffer.active.type === 'alternate' → 禁用补全
}
```

**Prompt 启发式检测**:

```typescript
// 匹配常见 shell 提示符结尾
const PROMPT_ENDINGS = [
  /\$\s$/,          // bash: "user@host:~$ "
  /#\s$/,           // root: "root@host:~# "
  /%\s$/,           // zsh: "user% "
  />\s$/,           // fish/powershell: "> "
  />>>\s$/,         // Python REPL
  /\)\s*\$\s$/,     // conda: "(base) user@host:~$ "
];
```

从当前行末尾向前扫描，找到最后一个匹配 prompt 模式的位置，其后内容即为当前命令。

### 1.4 边界情况

| 场景 | 处理方式 |
|------|---------|
| **密码提示** (无回显) | `onWriteParsed` 看不到变化，`command` 保持为空，不触发补全 |
| **vim/less/htop** | `buffer.active.type === 'alternate'` 立即禁用 |
| **Tab 补全** (shell 原生) | shell 回显后 `onWriteParsed` 重新提取，自动跟随 |
| **上下键历史** | 同上，buffer 反映 shell 实际显示的历史命令 |
| **Ctrl+R 反向搜索** | 检测搜索模式（`(reverse-i-search)`），暂停补全 |
| **多行命令** (反斜杠续行) | v1 仅追踪当前光标行，后续版本支持多行 |
| **自定义 PS1** | 启发式匹配覆盖 95% 场景；后续可加 OSC 133 精确检测 |

## 二、Ghost Text 渲染

### 2.1 设计

在终端光标之后渲染半透明的补全建议文本（ghost text），类似 IDE 的代码补全预览。

利用 xterm.js **Decoration API** — 已在 [useKeywordHighlight.ts:44-78](src/composables/useKeywordHighlight.ts#L44-L78) 中验证可用（`allowProposedApi: true` 已在 [useTerminal.ts:44](src/composables/useTerminal.ts#L44) 启用）。

### 2.2 实现

**新建文件**: `src/composables/useTerminalGhostText.ts` (~150 行)

```typescript
export function useTerminalGhostText(
  getTerminal: () => Terminal | null,
) {
  let currentDecoration: IDecoration | null = null;
  let currentMarker: IMarker | null = null;

  /** 在光标位置显示 ghost text */
  function show(text: string) {
    clear();
    const terminal = getTerminal();
    if (!terminal || !text) return;

    // 在光标行注册 marker (offset=0 表示当前光标行)
    currentMarker = terminal.registerMarker(0);
    if (!currentMarker) return;

    const cursorX = terminal.buffer.active.cursorX;

    currentDecoration = terminal.registerDecoration({
      marker: currentMarker,
      x: cursorX,
      width: text.length,
    });

    if (currentDecoration) {
      currentDecoration.onRender((el) => {
        el.textContent = text;
        // 从当前终端主题获取前景色，降低透明度以适配不同主题
        const fg = terminal.options.theme?.foreground || "#ffffff";
        el.style.color = fg;
        el.style.opacity = "0.35";
        el.style.fontFamily = terminal.options.fontFamily || "monospace";
        el.style.fontSize = `${terminal.options.fontSize}px`;
        el.style.pointerEvents = "none";
        el.style.whiteSpace = "pre";
        el.style.overflow = "visible";
        el.style.zIndex = "1";
      });
    }
  }

  /** 显示加载中占位符 */
  function showLoading() {
    show("...");
  }

  /** 清除当前 ghost text */
  function clear() {
    currentDecoration?.dispose();
    currentMarker?.dispose();
    currentDecoration = null;
    currentMarker = null;
  }

  function dispose() { clear(); }

  return { show, clear, dispose };
}
```

### 2.3 渲染策略

- **主方案**: xterm.js Decoration API (`registerMarker` + `registerDecoration` + `onRender`)
- **备选方案**: 如果 Decoration 无法正确显示多字符文本，使用绝对定位 `<div>` 叠加在终端容器内
  - 位置: `cursorX * cellWidth`, `cursorY * cellHeight`
  - cellWidth/cellHeight 通过 `terminal._core._renderService.dimensions` 获取

### 2.4 刷新时机

Ghost text 需要在以下时刻重建/清除：

| 事件 | 动作 |
|------|------|
| 新建议到达 | `clear()` → `show(newSuggestion)` |
| 用户输入任意字符 | `clear()`（等待新建议） |
| 用户按 Escape | `clear()` |
| 用户按 Tab 接受 | `clear()`，写入命令 |
| 终端接收远程输出 | `clear()`（buffer 已变化） |
| 终端滚动/resize | Decoration API 自动处理 |

/// AI 自动补全：根据部分命令文本返回候选补全列表
#[tauri::command]
pub async fn ai_autocomplete(
    state: State<'_, AppState>,
    context: AutocompleteContext,
) -> Result<Vec<String>, String> {
    let system = format!(
        "You are a shell autocomplete engine. Given a partial command and context, \
         suggest 1-5 possible complete commands.\n\
         Context: OS={}, Shell={}, CWD={}\n\
         Rules:\n\
         - Output ONLY a JSON array of complete command strings\n\
         - Rank by likelihood, most likely first\n\
         - Be concise, max 5 suggestions\n\
         - No explanation, no markdown, only JSON array",
        context.os.as_deref().unwrap_or("unknown"),
        context.shell.as_deref().unwrap_or("unknown"),
        context.cwd.as_deref().unwrap_or("~"),
    );

    let user_msg = if context.recent_commands.is_empty() {
        format!("Complete: {}", context.partial_command)

        format!(
            "Recent commands: {}\nComplete: {}",
            context.recent_commands.join(", "),
            context.partial_command,
        )
    };

    // 本地 AI 优先策略
    let response = autocomplete_with_local_first(&state, &system, &user_msg).await?;

    // 解析 JSON 数组
    parse_suggestions(&response)
}

async fn autocomplete_with_local_first(
    state: &AppState,
    system: &str,
    user_message: &str,
) -> Result<String, String> {
    // 1. 检查 llama-server 是否在运行
    let server = state.llama_server.read().await;
    if let Some(port) = server.port {
        drop(server);
        // 直接调用本地模型，无需 DB 查询
        return call_ai_provider(state, "local", "", None, "", system, user_message, Some(256)).await;
    }
    drop(server);

    // 2. 回退到默认 Provider
    let provider_info = state.db.with_conn(|conn| {
        conn.query_row(
            "SELECT id, provider_type, api_key_enc, api_base_url, model
             FROM ai_providers WHERE is_default = 1 LIMIT 1",
            [], |row| Ok((
                row.get::<_, String>(0)?,
                row.get::<_, String>(1)?,
                row.get::<_, Option<Vec<u8>>>(2)?,
                row.get::<_, Option<String>>(3)?,
                row.get::<_, String>(4)?,
            )),
        )
    }).map_err(|_| "no AI provider available for autocomplete".to_string())?;

    let (pid, provider_type, api_key_enc, api_base_url, model) = provider_info;
    let api_key = resolve_api_key(state, &pid, api_key_enc);

    call_ai_provider(
        state, &provider_type, &api_key, api_base_url.as_deref(),
        &model, system, user_message, Some(256),
    ).await
}

## 四、补全编排 Composable

### 4.1 设计

中枢 composable，将命令追踪、AI 请求、ghost text 渲染、弹窗控制串联起来。

### 4.2 实现

**新建文件**: `src/composables/useTerminalAutocomplete.ts` (~250 行)

```typescript
export function useTerminalAutocomplete(
  getTerminal: () => Terminal | null,
  sessionId: Ref<string>,
  commandState: Ref<CommandLineState>,
) {
  const suggestion = ref<string | null>(null);     // 当前 ghost text
  const suggestions = ref<string[]>([]);           // 全部候选
  const selectedIndex = ref(0);                    // 当前选中索引
  const popupVisible = ref(false);
  const loading = ref(false);

  const settingsStore = useSettingsStore();
  const ghostText = useTerminalGhostText(getTerminal);

  // LRU 缓存 (key: partialCommand+cwd, capacity: 100)
  const cache = new Map<string, string[]>();

  let debounceTimer: ReturnType<typeof setTimeout> | null = null;
  let requestId = 0;   // 单调递增，用于忽略过时响应

  // 监听命令变化
  watch(() => commandState.value.command, (cmd) => {
    if (!settingsStore.autocompleteEnabled) return;
    if (!commandState.value.atPrompt) return;

    // 清除当前建议
    dismiss();

    // 长度不够，不触发
    if (cmd.length < settingsStore.autocompleteMinChars) return;

    // Debounce
    if (debounceTimer) clearTimeout(debounceTimer);
    debounceTimer = setTimeout(() => fetchSuggestions(cmd), settingsStore.autocompleteDebounceMs);
  });

  async function fetchSuggestions(partialCommand: string) {
    const currentId = ++requestId;
    const cacheKey = `${partialCommand}|${/* cwd */ ""}`;

    // 检查缓存
    if (cache.has(cacheKey)) {
      applySuggestions(cache.get(cacheKey)!, partialCommand, currentId);
      return;
    }

    loading.value = true;
    try {
      const result = await tauriInvoke<string[]>("ai_autocomplete", {
        context: {
          partialCommand,
          os: /* from session metadata */ undefined,
          shell: undefined,
          cwd: undefined,
          recentCommands: /* recent command history buffer */ [],
        },
      });

      if (requestId !== currentId) return; // 过时响应，丢弃

      // 缓存结果
      if (cache.size >= 100) {
        const firstKey = cache.keys().next().value;
        cache.delete(firstKey);
      }
      cache.set(cacheKey, result);

      applySuggestions(result, partialCommand, currentId);
    } catch {
      // 静默失败 — 补全是可选增强
    } finally {
      loading.value = false;
    }
  }

  function applySuggestions(items: string[], partialCommand: string, id: number) {
    if (requestId !== id) return;
    suggestions.value = items;
    selectedIndex.value = 0;

    if (items.length > 0) {
      // 计算补全后缀: 完整命令 - 已输入部分
      const full = items[0];
      const suffix = full.startsWith(partialCommand) ? full.slice(partialCommand.length) : full;
      suggestion.value = suffix;
      ghostText.show(suffix);
    }
  }

  /** Tab 接受当前 ghost text */
  function accept() {
    if (!suggestion.value) return;
    const terminal = getTerminal();
    if (!terminal) return;

    // 写入补全后缀到终端（不追加 \n，用户自行回车）
    const bytes = new TextEncoder().encode(suggestion.value);
    const writeCmd = sessionId.value.startsWith("local-") ? "local_pty_write" : "ssh_write";
    tauriInvoke(writeCmd, {
      sessionId: sessionId.value,
      data: Array.from(bytes),
    }).catch(() => {});

    dismiss();
  }

  /** Escape 或继续输入 → 清除建议 */
  function dismiss() {
    suggestion.value = null;
    suggestions.value = [];
    popupVisible.value = false;
    ghostText.clear();
    if (debounceTimer) { clearTimeout(debounceTimer); debounceTimer = null; }
  }

  /** Ctrl+Space → 显示多选弹窗 */
  function showPopup() { if (suggestions.value.length > 0) popupVisible.value = true; }

  /** Alt+] → 切换到下一个建议 */
  function nextSuggestion() { /* cycle selectedIndex, update ghost text */ }

  /** Alt+[ → 切换到上一个建议 */
  function prevSuggestion() { /* cycle selectedIndex backward */ }

  function dispose() { dismiss(); ghostText.dispose(); }

  return {
    suggestion, suggestions, selectedIndex, popupVisible, loading,
    accept, dismiss, showPopup, nextSuggestion, prevSuggestion, dispose,
  };
}
```

### 6.1 设计

补全相关快捷键需要在 `attachCustomKeyEventHandler` 中处理（在 xterm.js 处理按键之前拦截），不在全局 `useShortcuts.ts` 中处理。

### 6.2 修改文件

**修改**: [useTerminal.ts:67-93](src/composables/useTerminal.ts#L67-L93)

在现有 Copy/Paste 处理之后，添加补全快捷键拦截：

```typescript
terminal.attachCustomKeyEventHandler((ev) => {
  if (ev.type !== "keydown") return true;

  // ... 现有 Copy/Paste 处理 ...

  // === 补全快捷键 ===
  const autocomplete = getAutocomplete(); // 从外部传入的 autocomplete composable

  // Tab: 接受 ghost text（仅当有建议时）
  if (ev.key === "Tab" && !ev.shiftKey && !ev.ctrlKey && !ev.metaKey && !ev.altKey) {
    if (autocomplete?.suggestion.value) {
      autocomplete.accept();
      return false; // 阻止 Tab 发送到终端
    }
    return true; // 无建议时正常 Tab（shell 补全）
  }

  // Escape: 清除建议
  if (ev.key === "Escape") {
    if (autocomplete?.suggestion.value || autocomplete?.popupVisible.value) {
      autocomplete.dismiss();
      return false;
    }
    return true;
  }

  // Ctrl+Space: 显示/触发补全弹窗
  if (ev.key === " " && ev.ctrlKey && !ev.shiftKey && !ev.metaKey) {
    autocomplete?.showPopup();
    return false;
  }

  // Alt+]: 下一个建议
  if (ev.key === "]" && ev.altKey) {
    autocomplete?.nextSuggestion();
    return false;
  }

  // Alt+[: 上一个建议
  if (ev.key === "[" && ev.altKey) {
    autocomplete?.prevSuggestion();
    return false;
  }

  return true;
});
```

**关键**: Tab 键冲突处理 — 当 ghost text 可见时 Tab 接受建议，不可见时 Tab 传递给 shell 做原生补全。

## 九、请求取消与超时

### 9.1 问题

用户快速输入 `git c` → `git ch` → `git che` 时，debounce 只能合并连续输入，但如果 debounce 触发后用户继续输入，前一个 AI 请求仍在执行。本地模型会浪费 CPU/GPU 做无用推理，云端请求也可能在用户输完命令后才返回，ghost text 反而干扰。

### 9.2 前端取消

**修改**: `useTerminalAutocomplete.ts`

```typescript
let abortController: AbortController | null = null;

async function fetchSuggestions(partialCommand: string) {
  // 取消上一个正在进行的请求
  if (abortController) abortController.abort();
  abortController = new AbortController();

  const currentId = ++requestId;
  // ... 缓存检查 ...

  loading.value = true;
  try {
    const result = await tauriInvoke<string[]>("ai_autocomplete", {
      context: { partialCommand, ... },
    });
    if (requestId !== currentId) return; // 过时响应
    // ... apply suggestions ...
  } catch (e) {
    // AbortError 或 AI 失败 — 静默忽略
  } finally {
    loading.value = false;
  }
}
```

## 十、AI 响应格式容错

### 10.1 问题

LLM 返回格式不可靠，可能包含：
- 标准 JSON: `["git checkout", "git cherry-pick"]`
- Markdown 包裹: `` ```json\n["git checkout"]\n``` ``
- 带前缀文本: `Here are suggestions:\n["git checkout"]`
- 换行分隔: `git checkout\ngit cherry-pick`
- 单条字符串: `"git checkout"`

### 10.2 多层降级解析

**修改**: `src-tauri/src/commands/ai.rs`

```rust
fn parse_suggestions(raw: &str) -> Result<Vec<String>, String> {
    let trimmed = raw.trim();

    // 1. 直接 JSON 数组
    if let Ok(arr) = serde_json::from_str::<Vec<String>>(trimmed) {
        return Ok(arr.into_iter().filter(|s| !s.is_empty()).take(5).collect());
    }

    // 2. 提取 markdown code fence 中的 JSON
    if let Some(json_block) = extract_code_fence(trimmed) {
        if let Ok(arr) = serde_json::from_str::<Vec<String>>(json_block) {
            return Ok(arr.into_iter().filter(|s| !s.is_empty()).take(5).collect());
        }
    }

    // 3. 正则提取 JSON 数组 (忽略前后文本)
    if let Some(start) = trimmed.find('[') {
        if let Some(end) = trimmed.rfind(']') {
            let slice = &trimmed[start..=end];
            if let Ok(arr) = serde_json::from_str::<Vec<String>>(slice) {
                return Ok(arr.into_iter().filter(|s| !s.is_empty()).take(5).collect());
            }
        }
    }

    // 4. 按行分割 (每行一个命令)
    let lines: Vec<String> = trimmed
        .lines()
        .map(|l| l.trim().trim_start_matches(|c: char| c == '-' || c == '*' || c.is_ascii_digit() || c == '.').trim().to_string())
        .filter(|l| !l.is_empty() && !l.starts_with("```") && !l.starts_with("Here"))
        .take(5)
        .collect();
    if !lines.is_empty() {
        return Ok(lines);
    }

    // 5. 单条字符串
    if let Ok(single) = serde_json::from_str::<String>(trimmed) {
        return Ok(vec![single]);
    }

    // 6. 原文当作单条命令
    if !trimmed.is_empty() && trimmed.len() < 200 {
        return Ok(vec![trimmed.to_string()]);
    }

    Ok(vec![])
}

fn extract_code_fence(text: &str) -> Option<&str> {
    let start = text.find("```")? + 3;
    let after_lang = text[start..].find('\n')? + start + 1;
    let end = text[after_lang..].find("```")? + after_lang;
    Some(text[after_lang..end].trim())
}
```

## 十四、弹窗定位降级方案

### 14.1 问题

弹窗定位使用了 xterm.js 私有 API `_core._renderService.dimensions`，版本升级可能 break。

### 14.2 降级策略

```typescript
function getCellDimensions(terminal: Terminal): { width: number; height: number } {
  // 方案 1: 私有 API (最精确)
  try {
    const dims = (terminal as any)._core._renderService.dimensions;
    if (dims?.css?.cell?.width) {
      return { width: dims.css.cell.width, height: dims.css.cell.height };
    }
  } catch { /* fall through */ }

  // 方案 2: 用 canvas 测量 (降级)
  const canvas = document.createElement("canvas");
  const ctx = canvas.getContext("2d");
  if (ctx) {
    const fontFamily = terminal.options.fontFamily || "monospace";
    const fontSize = terminal.options.fontSize || 14;
    ctx.font = `${fontSize}px ${fontFamily}`;
    const metrics = ctx.measureText("M");
    return {
      width: metrics.width,
      height: fontSize * 1.2, // 近似行高
    };
  }

  // 方案 3: 硬编码默认值
  return { width: 8.4, height: 17 };
}
```

## 十五、安全过滤 — 敏感命令不发送给 AI

### 15.1 问题

CLAUDE.md 硬性规定："NEVER send credentials to AI providers — only send command text and server metadata"。
用户输入的 partial command 本身可能包含敏感信息：

```bash
mysql -u root -p MySecret123
curl -u admin:password https://...
export API_KEY=sk-xxxx
sshpass -p 'hunter2' ssh user@host
```

### 15.2 设计

**修改**: `src/composables/useTerminalAutocomplete.ts` — `fetchSuggestions` 入口处添加安全检查

```typescript
const SENSITIVE_PATTERNS = [
  /(-p|--password|--token|--secret)\s+\S+/i,
  /export\s+\w*(KEY|SECRET|TOKEN|PASSWORD|CREDENTIAL)\w*\s*=/i,
  /sshpass\s+-p\s+/i,
  /curl\s+.*-u\s+\w+:/i,
  /mysql\s+.*-p\S+/i,
];

function containsSensitive(command: string): boolean {
  return SENSITIVE_PATTERNS.some(p => p.test(command));
}
```

- **云端 Provider**: 检测到敏感模式时**跳过本次补全请求**，不发送给 AI
- **本地模型 (llama-server)**: 数据不出本机，**不受此限制**
- 通过 `autocompletePreferLocal` 设置 + 后端 `is_local` 标记联动

### 15.3 后端联动

**修改**: `src-tauri/src/commands/ai.rs` — `ai_autocomplete` 返回值增加 `is_local` 标记

```rust
#[derive(Serialize)]
#[serde(rename_all = "camelCase")]
pub struct AutocompleteResult {
    pub suggestions: Vec<String>,
    pub is_local: bool,  // 本次请求是否走的本地模型
}
```

前端据此决定是否对敏感命令放行。

## 十七、useTerminal 循环依赖解决

### 17.1 问题

键盘快捷键需在 `useTerminal.ts` 的 `attachCustomKeyEventHandler` 中拦截（在 xterm.js 处理前）。但 `useTerminalAutocomplete` 依赖 `getTerminal()`，而 `useTerminal` 又需要引用 autocomplete 状态来判断 Tab 是否拦截。

### 17.2 方案：Ref 注入

`useTerminal` 接受一个可选的 `autocompleteRef` 参数，初始为 `null`，在 `TerminalView.vue` mount 后赋值：

**修改**: `src/composables/useTerminal.ts`

```typescript
export interface TerminalOptions {
  onShellReady?: (sessionId: string) => Promise<void>;
  /** 延迟注入的 autocomplete API，解决循环依赖 */
  autocompleteRef?: Ref<AutocompleteAPI | null>;
}

export interface AutocompleteAPI {
  suggestion: Ref<string | null>;
  popupVisible: Ref<boolean>;
  accept: () => void;
  dismiss: () => void;
  showPopup: () => void;
  nextSuggestion: () => void;
  prevSuggestion: () => void;
}
```

`attachCustomKeyEventHandler` 中通过 `options?.autocompleteRef?.value` 访问：

```typescript
terminal.attachCustomKeyEventHandler((ev) => {
  // ... Copy/Paste ...

  const ac = options?.autocompleteRef?.value;
  if (ac && ev.key === "Tab" && !ev.shiftKey && !ev.ctrlKey && !ev.metaKey && !ev.altKey) {
    if (ac.suggestion.value) { ac.accept(); return false; }
  }
  // ...
});
```

**TerminalView.vue 初始化顺序**:

```typescript
const autocompleteRef = ref<AutocompleteAPI | null>(null);
const { mount, getTerminal, ... } = useTerminal(sessionIdRef, { onShellReady, autocompleteRef });

// mount 之后初始化 autocomplete
onMounted(async () => {
  await mount(containerRef.value!);
  const commandTracker = useCommandTracker(getTerminal, sessionIdRef);
  commandTracker.init();
  const autocomplete = useTerminalAutocomplete(getTerminal, sessionIdRef, commandTracker.state);
  autocompleteRef.value = autocomplete;  // 注入到 useTerminal
});
```

function computePopupPosition(terminal: Terminal, containerEl: HTMLElement) {
  // ... 基础计算 ...

  // Viewport boundary clamping
  const popupWidth = 400;  // 预估最大宽度
  const popupHeight = suggestions.length * 32 + 8;  // 预估高度
