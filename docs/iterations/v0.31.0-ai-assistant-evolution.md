## 背景

### 当前 AI 能力评估

Termex 的 AI 目前处于 **"工具级"** 能力阶段：

| 功能 | 状态 | 模块 |
|------|------|------|
| 危险命令检测 (40+ 正则规则) | 已完成 | `src-tauri/src/ai/danger.rs` |
| 命令解释 (流式输出) | 已完成 | `ai_explain_command` |
| NL2Cmd (自然语言 -> 命令) | 已完成 | `ai_nl2cmd` |
| 终端内联智能补全 (ghost text) | 已完成 | `useTerminalAutocomplete.ts` |
| 12 种 Provider 支持 | 已完成 | `src-tauri/src/ai/provider.rs` |
| 本地 AI 引擎 (llama-server) | 已完成 | `src-tauri/src/local_ai/` |
| **上下文感知对话** | **未实现** | AI 不知道终端当前状态 |
| **错误自动诊断** | **未实现** | 命令失败后无自动分析 |
| **多轮排障对话** | **未实现** | 无完整对话 UI |
| **命令编排 (Playbook)** | **未实现** | 无多步骤执行引擎 |
| **异常检测联动** | **未实现** | 监控数据未接入 AI |
| **会话总结报告** | **未实现** | 无操作回顾功能 |

### 核心痛点

1. **AI 是"无头"的** — 不知道用户在哪台服务器、什么目录、刚执行了什么命令
2. **错误处理靠人** — `Permission denied` 或 `disk full` 只能用户自己排查
3. **对话不连续** — 每次交互是独立的，无法追问"上一个错误怎么修？"
4. **复杂操作无引导** — "配置 nginx 反向代理"需要用户自己拆解成多个命令

### 目标

将 AI 从"工具级"升级到"助手级" — context-aware、proactive、multi-turn，深度集成 SSH 终端。

```typescript
import type { Terminal } from "@xterm/xterm";
import type { TerminalContext, ContextOptions, DEFAULT_CONTEXT_OPTIONS } from "@/types/aiContext";
import type { CommandLineState, TerminalMode } from "@/types/commandTracker";
import { useSessionStore } from "@/stores/sessionStore";
import { useServerStore } from "@/stores/serverStore";

export function useAiContext(
  getTerminal: () => Terminal | null,
  sessionId: Ref<string>,
  commandState: Ref<CommandLineState>,
  terminalMode: Ref<TerminalMode>,
) {
  /** Extracts current terminal context for AI injection. */
  function captureContext(options: ContextOptions = DEFAULT_CONTEXT_OPTIONS): TerminalContext | null {
    const terminal = getTerminal();
    if (!terminal) return null;

    const sessionStore = useSessionStore();
    const serverStore = useServerStore();
    const session = sessionStore.sessions.get(sessionId.value);
    if (!session) return null;

    const server = serverStore.servers.find(s => s.id === session.serverId);

    // Extract recent output from terminal buffer
    const recentOutput = options.includeOutput
      ? extractRecentLines(terminal, options.outputLines)
      : "";

    return {
      server: options.includeServer ? {
        hostname: server?.host ?? "unknown",
        os: server?.os ?? "unknown",
        username: server?.username ?? "unknown",
        connectionChain: buildChainDescription(session),
      } : { hostname: "", os: "", username: "", connectionChain: "" },
      shell: options.includeShell ? {
        cwd: extractCwd(terminal),
        lastCommand: commandState.value.command,
        lastExitCode: null,  // Populated by error detection
        terminalMode: terminalMode.value,
      } : { cwd: "", lastCommand: "", lastExitCode: null, terminalMode: "unknown" },
      recentOutput,
      capturedAt: new Date().toISOString(),
    };
  }

  return { captureContext };
}

/** Reads last N lines from xterm.js active buffer. */
function extractRecentLines(terminal: Terminal, count: number): string {
  const buffer = terminal.buffer.active;
  const lines: string[] = [];
  const start = Math.max(0, buffer.cursorY + buffer.baseY - count);
  const end = buffer.cursorY + buffer.baseY;

  for (let i = start; i <= end; i++) {
    const line = buffer.getLine(i);
    if (line) {
      lines.push(line.translateToString(true));
    }
  }
  return lines.join("\n");
}
```

### 1.3 Context → System Prompt 构建

**新建文件**: `src-tauri/src/ai/context.rs` (~120 行)

将前端传入的 context 格式化为 system prompt 注入 AI 请求。

```rust
use serde::Deserialize;

/// Terminal context sent from frontend for AI conversations.
#[derive(Debug, Clone, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct TerminalContext {
    pub server: ServerContext,
    pub shell: ShellContext,
    pub recent_output: String,
    pub captured_at: String,
}

#[derive(Debug, Clone, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ServerContext {
    pub hostname: String,
    pub os: String,
    pub username: String,
    pub connection_chain: String,
}

#[derive(Debug, Clone, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ShellContext {
    pub cwd: String,
    pub last_command: String,
    pub last_exit_code: Option<i32>,
    pub terminal_mode: String,
}

/// Builds a system prompt with terminal context injected.
pub fn build_context_prompt(base_prompt: &str, ctx: &TerminalContext) -> String {
    let mut prompt = String::with_capacity(base_prompt.len() + 512);
    prompt.push_str(base_prompt);
    prompt.push_str("\n\n--- Terminal Context ---\n");

    if !ctx.server.hostname.is_empty() {
        prompt.push_str(&format!(
            "Server: {}@{} ({})\nConnection: {}\n",
            ctx.server.username, ctx.server.hostname,
            ctx.server.os, ctx.server.connection_chain,
        ));
    }

    if !ctx.shell.cwd.is_empty() {
        prompt.push_str(&format!("CWD: {}\n", ctx.shell.cwd));
    }
    if !ctx.shell.last_command.is_empty() {
        prompt.push_str(&format!("Last command: {}\n", ctx.shell.last_command));
    }
    if let Some(code) = ctx.shell.last_exit_code {
        prompt.push_str(&format!("Exit code: {}\n", code));
    }

    if !ctx.recent_output.is_empty() {
        prompt.push_str(&format!(
            "\n--- Recent Terminal Output ---\n{}\n--- End Output ---\n",
            ctx.recent_output
        ));
    }

    prompt
}
```

**安全规则**: `TerminalContext` 不包含任何凭据字段。Server password / API key / private key 绝不会被注入到 AI prompt。`ServerContext` 仅包含 hostname、OS、username — 与 v0.5.0 的安全规则一致。

```rust
/// Input for multi-turn AI chat.
#[derive(Debug, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ChatInput {
    pub session_id: String,
    pub message: String,
    pub include_context: bool,
    pub context: Option<crate::ai::context::TerminalContext>,
    /// Previous messages for multi-turn context (last N turns).
    pub history: Vec<ChatHistoryEntry>,
}

#[derive(Debug, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ChatHistoryEntry {
    pub role: String,     // "user" | "assistant"
    pub content: String,
}

/// Multi-turn AI chat with optional terminal context injection.
#[tauri::command]
pub async fn ai_chat(
    app: AppHandle,
    state: State<'_, AppState>,
    input: ChatInput,
    request_id: String,
) -> Result<(), String> {
    let provider_config = get_default_provider(&state)?;
    let provider = create_provider(&provider_config)?;

    // Build system prompt with context
    let base_prompt = "You are Termex AI, an expert Linux/Unix operations assistant. \
        You help users diagnose issues, write commands, and manage servers. \
        When suggesting commands, wrap each in a ```bash code block. \
        Be concise and actionable.";

    let system_prompt = if input.include_context {
        if let Some(ref ctx) = input.context {
            crate::ai::context::build_context_prompt(base_prompt, ctx)
        } else {
            base_prompt.to_string()
        }
    } else {
        base_prompt.to_string()
    };

    // Build conversation messages
    let mut messages = Vec::new();
    for entry in &input.history {
        messages.push(format!("{}: {}", entry.role, entry.content));
    }
    messages.push(format!("user: {}", input.message));
    let full_message = messages.join("\n\n");

    let request = AiRequest {
        system_prompt,
        user_message: full_message,
        max_tokens: provider_config.max_tokens as u32,
    };

    let event_name = format!("ai://chat/{}", request_id);
    let app_clone = app.clone();
    provider
        .stream(
            &request,
            Box::new(move |chunk| {
                let _ = app_clone.emit(&event_name, &chunk);
            }),
        )
        .await
        .map_err(|e| e.to_string())
}
```

/** Sends a chat message with optional terminal context. */
async function sendChat(
  sessionId: string,
  message: string,
  context: TerminalContext | null,
): Promise<void> {
  const conv = getConversation(sessionId);
  const requestId = crypto.randomUUID();

  // Add user message
  conv.messages.push({
    id: crypto.randomUUID(),
    role: "user",
    content: message,
    timestamp: new Date().toISOString(),
  });

  // Add context message (collapsed in UI)
  if (context) {
    conv.messages.push({
      id: crypto.randomUUID(),
      role: "context",
      content: "Terminal context attached",
      timestamp: new Date().toISOString(),
      context,
    });
  }

  // Add streaming assistant message placeholder
  const assistantMsg: ChatMessage = {
    id: requestId,
    role: "assistant",
    content: "",
    timestamp: new Date().toISOString(),
    streaming: true,
    commands: [],
  };
  conv.messages.push(assistantMsg);

  // Build history (last 10 turns for API context)
  const history = conv.messages
    .filter(m => m.role === "user" || m.role === "assistant")
    .slice(-10)
    .map(m => ({ role: m.role, content: m.content }));

  // Listen for streaming chunks
  const unlisten = await tauriListen<AiChunk>(
    `ai://chat/${requestId}`,
    (chunk) => {
      const target = conv.messages.find(m => m.id === requestId);
      if (target) {
        target.content += chunk.text;
        if (chunk.done) {
          target.streaming = false;
          target.commands = extractCommands(target.content);
        }
      }
    },
  );

  try {
    await tauriInvoke("ai_chat", {
      input: {
        sessionId,
        message,
        includeContext: !!context,
        context,
        history: history.slice(0, -1),  // Exclude current assistant placeholder
      },
      requestId,
    });
  } catch (err) {
    assistantMsg.content = String(err);
    assistantMsg.streaming = false;
  }

  unlisten();
}

/** Extracts bash commands from AI markdown response. */
function extractCommands(content: string): ExtractedCommand[] {
  const commands: ExtractedCommand[] = [];
  const regex = /```(?:bash|sh|shell)?\n([\s\S]*?)```/g;
  let match: RegExpExecArray | null;
  while ((match = regex.exec(content)) !== null) {
    const cmd = match[1].trim();
    if (cmd) {
      commands.push({
        command: cmd,
        description: "",
        dangerous: false,
      });
    }
  }
  return commands;
}

/** Clears conversation for a session (on disconnect or manual clear). */
function clearConversation(sessionId: string): void {
  conversations.value.delete(sessionId);
}

export interface DetectedError {
  /** The command that produced the error. */
  command: string;
  /** Error output lines from terminal. */
  errorOutput: string;
  /** Matched error pattern (for debugging). */
  matchedPattern: string;
  /** Severity: info (退出码非零), warning (known error), critical (data loss risk). */
  severity: "info" | "warning" | "critical";
  /** Timestamp of detection. */
  detectedAt: string;
}

/** Error patterns categorized by severity. */
const ERROR_PATTERNS: Array<{ pattern: RegExp; severity: DetectedError["severity"]; label: string }> = [
  // Critical (data loss / security risk)
  { pattern: /No space left on device/i, severity: "critical", label: "disk_full" },
  { pattern: /Read-only file system/i, severity: "critical", label: "readonly_fs" },
  { pattern: /Segmentation fault/i, severity: "critical", label: "segfault" },
  { pattern: /kernel panic/i, severity: "critical", label: "kernel_panic" },
  { pattern: /Out of memory/i, severity: "critical", label: "oom" },
  { pattern: /FATAL/i, severity: "critical", label: "fatal" },

  // Warning (common operational errors)
  { pattern: /command not found/i, severity: "warning", label: "cmd_not_found" },
  { pattern: /Permission denied/i, severity: "warning", label: "permission_denied" },
  { pattern: /No such file or directory/i, severity: "warning", label: "no_such_file" },
  { pattern: /Connection refused/i, severity: "warning", label: "conn_refused" },
  { pattern: /Connection timed out/i, severity: "warning", label: "conn_timeout" },
  { pattern: /Name or service not known/i, severity: "warning", label: "dns_fail" },
  { pattern: /Could not resolve hostname/i, severity: "warning", label: "dns_fail" },
  { pattern: /Authentication failure/i, severity: "warning", label: "auth_fail" },
  { pattern: /error:\s/i, severity: "warning", label: "generic_error" },
  { pattern: /E:\s+Unable to/i, severity: "warning", label: "apt_error" },
  { pattern: /Failed to start/i, severity: "warning", label: "service_fail" },

## 四、AI Command Orchestration: 命令编排引擎

### 4.1 设计理念

用户描述一个高级目标（如"部署 Node.js 应用到 nginx"），AI 将其分解为一系列有序命令步骤。用户可以逐步执行（点击每个步骤）或一键全部运行。每步执行后 AI 检查输出判断是否成功。

类似 Ansible Playbook，但完全交互式、即时生成。

### 4.2 Orchestration 数据模型

**修改文件**: `src/types/ai.ts`

```typescript
/** A step in an AI-generated command sequence. */
export interface OrchestratedStep {
  id: string;
  /** Step number (1-based). */
  stepNumber: number;
  /** Human-readable description. */
  description: string;
  /** The command to execute. */
  command: string;
  /** Step state. */
  status: "pending" | "running" | "success" | "failed" | "skipped";
  /** Terminal output after execution (for AI validation). */
  output?: string;
  /** AI validation result after execution. */
  validation?: string;
  /** Whether the command is dangerous (requires extra confirmation). */

  id: string;
  /** The user's original goal description. */
  goal: string;

  /** Overall status. */

### 4.3 Orchestration Backend

**新建文件**: `src-tauri/src/ai/orchestrator.rs` (~150 行)

```rust
use serde::{Deserialize, Serialize};
use crate::ai::provider::{AiProvider, AiRequest, AiError};

/// A parsed step from AI orchestration response.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ParsedStep {
    pub step_number: u32,
    pub description: String,
    pub command: String,
    pub dangerous: bool,
}

/// Parses AI response into structured command steps.
///
/// Expected AI output format:
/// ```
/// STEP 1: Install nginx
/// ```bash
/// sudo apt install -y nginx
/// ```
///
/// STEP 2: Create config file
/// ```bash
/// sudo tee /etc/nginx/sites-available/myapp > /dev/null <<'EOF'
/// server { ... }
/// EOF
/// ```
/// ```
pub fn parse_orchestration_response(content: &str) -> Vec<ParsedStep> {
    let mut steps = Vec::new();
    let step_re = regex::Regex::new(
        r"(?i)STEP\s+(\d+)[:\s]+(.+?)(?:\r?\n)```(?:bash|sh|shell)?\n([\s\S]*?)```"
    ).unwrap();

    for cap in step_re.captures_iter(content) {
        let step_num: u32 = cap[1].parse().unwrap_or(0);
        let desc = cap[2].trim().to_string();
        let cmd = cap[3].trim().to_string();

        // Check if command is dangerous (reuse DangerDetector)
        let danger = crate::ai::danger::DangerDetector::check(&cmd);

        steps.push(ParsedStep {
            step_number: step_num,
            description: desc,
            command: cmd,
            dangerous: danger.is_dangerous,
        });
    }
    steps
}

/// System prompt for orchestration requests.
pub const ORCHESTRATION_PROMPT: &str = "\
You are Termex AI, an expert operations assistant. \
The user will describe a goal. Generate a step-by-step command sequence. \
\n\nFormat EXACTLY as:\n\
STEP 1: <description>\n```bash\n<command>\n```\n\n\
STEP 2: <description>\n```bash\n<command>\n```\n\n\
Rules:\n\
- One command per step (pipe chains are OK)\n\
- Include error checking where appropriate (e.g., `set -e`)\n\
- Use sudo only when necessary\n\
- Prefer non-destructive commands\n\
- Maximum 15 steps";
```

### 4.4 Orchestration IPC

**修改文件**: `src-tauri/src/commands/ai.rs`

```rust
/// Orchestrate a command sequence for a complex goal.
#[tauri::command]
pub async fn ai_orchestrate(
    app: AppHandle,
    state: State<'_, AppState>,
    goal: String,
    context: Option<crate::ai::context::TerminalContext>,
    request_id: String,
) -> Result<(), String> {
    let provider_config = get_default_provider(&state)?;
    let provider = create_provider(&provider_config)?;

    let system_prompt = if let Some(ref ctx) = context {
        crate::ai::context::build_context_prompt(
            crate::ai::orchestrator::ORCHESTRATION_PROMPT, ctx
        )
    } else {
        crate::ai::orchestrator::ORCHESTRATION_PROMPT.to_string()
    };

    let request = AiRequest {
        system_prompt,
        user_message: goal,
        max_tokens: 4096,
    };

    let event_name = format!("ai://orchestrate/{}", request_id);
    let app_clone = app.clone();
    provider
        .stream(
            &request,
            Box::new(move |chunk| {
                let _ = app_clone.emit(&event_name, &chunk);
            }),
        )
        .await
        .map_err(|e| e.to_string())
}
```

## 八、安全合规

### 不发送到 AI 的信息 (CRITICAL)

- SSH 密码 / 私钥 / passphrase
- AI Provider API Key
- 数据库 master password
- Keychain 内容
- `.env` 文件内容 (如终端 `cat .env` 的输出出现在 buffer 中 — 当前版本不做过滤，v0.33 考虑)

### 发送到 AI 的信息

- 服务器 hostname、OS、username (已有先例: v0.5.0 的 explain/nl2cmd)
- 终端 buffer 文本 (用户可见的内容)
- 用户主动输入的消息
- 命令文本

### 用户控制

- 每次 chat 可选择是否附带 context (`include_context: bool`)
- Auto-diagnose 可全局关闭
- AI panel 关闭时不监听终端输出
- 清除对话历史按钮

## 十、已知限制与后续迭代

1. **Buffer 安全过滤**: 终端 buffer 可能包含敏感信息 (`cat /etc/shadow` 输出)。当前版本原样发送，v0.33 将增加敏感内容过滤层
2. **Exit code 检测**: 依赖 prompt 启发式（v0.26.0 的 `useCommandTracker`），精度约 85%。后续可通过 `trap 'echo $?' DEBUG` 注入提升
3. **Orchestration 验证**: 步骤间 AI 验证需要额外 API 调用（增加延迟和成本）。可选配置
4. **本地 AI 适配**: 小型本地模型 (7B) 的编排质量可能不佳。建议使用 13B+ 或 API 模型
5. **对话持久化**: 当前对话仅存内存，断开即清除。v0.33 可选持久化到 SQLCipher
6. **并发诊断**: 多个 tab 同时出错时共用同一 AI 请求队列，避免 API 过载
