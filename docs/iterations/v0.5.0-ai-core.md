# v0.5.0 -- AI Core: Danger Detection + Command Explanation

> Status: **Planned**
> Depends on: v0.4.0

## Goals

引入 AI 一等公民能力：危险命令实时拦截 + 命令智能解释，构建安全守卫层。

## Features

### AI Provider Abstraction
- [ ] ai/provider.rs -- Provider trait 定义
- [ ] ai/claude.rs -- Claude API 集成
- [ ] ai/openai.rs -- OpenAI API 集成
- [ ] ai/ollama.rs -- Ollama 本地模型集成
- [ ] API Key 加密存储（复用 AES-256-GCM）
- [ ] Provider CRUD + 默认 Provider 设置
- [ ] AiConfigTab.vue -- Provider 列表 + 添加/编辑表单
- [ ] commands/ai.rs -- ai_provider_list, add, update, delete, set_default, test

### Dangerous Command Detection
- [ ] 两层检测架构：
  - 第一层：本地正则匹配（`rm -rf /`, `dd`, `mkfs`, `chmod 777` 等 30+ 规则）
  - 第二层：AI 语义分析（可选，需 Provider 配置）
- [ ] DangerAlert.vue -- 危险命令拦截弹窗（确认/取消/永久跳过）
- [ ] 危险等级分级：warning（黄色）/ critical（红色）
- [ ] 白名单机制（用户确认后不再拦截）
- [ ] 事件推送：ai://danger/{session_id}

### Command Explanation
- [ ] 选中终端文本 → 右键 "Explain" 或 Ctrl+E
- [ ] AI 解释面板（侧边弹出）
- [ ] 流式输出（SSE / streaming response）
- [ ] 命令历史解释缓存

### Tests
- [ ] 危险命令正则匹配测试（30+ 测试用例）
- [ ] AI Provider trait mock 测试
- [ ] 流式响应解析测试
