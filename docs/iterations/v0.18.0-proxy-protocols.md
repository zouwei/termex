# V0.18.0 — SSH 代理连接协议支持

## Context

通过 SOCKS5 代理连接 SSH 服务器是常见需求。本迭代一次性补全 SSH 客户端常见的代理连接协议（SOCKS5 / SOCKS4 / HTTP CONNECT），对标 PuTTY / SecureCRT / Termius 的内置代理支持。

### 当前状态

- Termex 仅支持直连和 SSH ProxyJump（v0.12.0），不支持任何网络代理协议
- ConnectModal 有两个 tab：Authorization（认证信息）和 SSH Tunnel（跳板机）

### 设计决策（用户确认）

- 代理配置采用**独立代理表 + 引用**模式（类似 Termius），在 Settings 中全局管理代理列表
- ConnectModal 新增独立的 **"Proxy" tab**（与 Authorization / SSH Tunnel 平级）
- 每个 SSH 连接可单独选择一个代理，多个连接可共享同一代理
- SOCKS4/4a 与 SOCKS5/HTTP 一起实现（tokio-socks 原生支持）

## 一、代理协议范围

| 协议 | 说明 | 认证 | 默认端口 |
|------|------|------|----------|
| **SOCKS5** (RFC 1928) | 最通用，支持 IPv4/IPv6/域名 | 无认证 / 用户名密码 | 1080 |
| **HTTP CONNECT** | 企业环境最常见，HTTP 代理隧道 | 无认证 / Basic Auth | 8080 |
| **SOCKS4** | 仅 IPv4 IP | User ID | 1080 |
| **SOCKS4a** | SOCKS4 + 域名解析在代理端 | User ID | 1080 |

### 不实现

| 协议 | 原因 |
|------|------|
| HTTP NTLM/Negotiate | NTLM 实现复杂，使用场景有限 |
| Telnet 代理 | 已过时 |
| ProxyCommand | GUI 应用中执行任意 shell 命令有安全风险 |

## 三、架构设计

### 3.1 代理层：TCP 传输替换

代理层位于 TCP 连接与 SSH 握手之间，产出 `AsyncRead + AsyncWrite` 流交给 russh，与 SSH 逻辑完全解耦。

```
直连：     TcpStream → SSH Handshake
SOCKS5：   TcpStream → SOCKS5 Handshake → [Tunnel] → SSH Handshake
HTTP：     TcpStream → HTTP CONNECT → [Tunnel] → SSH Handshake
ProxyJump：SSH Session → direct-tcpip Channel → SSH Handshake

组合：     TcpStream → SOCKS5 → Bastion SSH → direct-tcpip → Target SSH
```

### 3.2 Rust 代理模块

新模块 `src-tauri/src/ssh/proxy.rs`：

```rust
pub struct ProxyConfig {
    pub proxy_type: ProxyType,
    pub host: String,
    pub port: u16,
    pub username: Option<String>,
    pub password: Option<String>,
}

/// Connects to target through a proxy, returns async stream.
pub async fn connect_via_proxy(
    proxy: &ProxyConfig,
    target_host: &str,
    target_port: u16,
) -> Result<Box<dyn AsyncRead + AsyncWrite + Unpin + Send>, SshError>
```

- SOCKS5/4/4a：`tokio-socks` crate（成熟的异步 SOCKS 客户端）
- HTTP CONNECT：手工实现（~50 行，发送 `CONNECT host:port HTTP/1.1`，解析 `200` 响应）

### 3.3 SshSession 扩展

```rust
/// Connects via network proxy (SOCKS/HTTP) then performs SSH handshake.
pub async fn connect_via_network_proxy(
    proxy: &ProxyConfig, host: &str, port: u16,
) -> Result<Self, SshError>
```

内部：`connect_via_proxy()` → `russh::client::connect_stream()` SSH 握手。

### 3.4 ssh_connect 四分支

```rust
let session = match (network_proxy, bastion) {
    (Some(proxy), Some(bastion)) => {
        // 代理 → 跳板机 → 目标
        let bastion_session = SshSession::connect_via_network_proxy(&proxy, bastion_host, bastion_port).await?;
        // 认证跳板机...
        SshSession::connect_via_proxy(bastion_session.handle(), target_host, target_port).await?
    }
    (Some(proxy), None) => {
        // 代理 → 目标
        SshSession::connect_via_network_proxy(&proxy, target_host, target_port).await?
    }
    (None, Some(bastion)) => { /* 现有 ProxyJump 逻辑不变 */ }
    (None, None) => { /* 现有直连逻辑不变 */ }
};
```

## 四、UI 设计

### 4.1 Settings → Proxies 全局管理面板

在 Settings 中新增 **"Proxies"** 标签页，管理所有代理配置：

```
┌──────────────────────────────────────────────────────┐
│  Proxies                              [+ Add Proxy]  │
│                                                      │
│  ┌─────────┬──────────┬───────────────────┬────────┐ │
│  │ Name    │ Type     │ Host:Port          │        │ │
│  ├─────────┼──────────┼───────────────────┼────────┤ │
│  │ Office  │ SOCKS5   │ proxy.corp:1080    │ ✏ ✕   │ │
│  │ Home    │ HTTP     │ squid.local:8080   │ ✏ ✕   │ │
│  └─────────┴──────────┴───────────────────┴────────┘ │
│                                                      │
│  Used by: 3 server(s)                                │
└──────────────────────────────────────────────────────┘
```

点击 "Add Proxy" 或 ✏ 弹出行内编辑表单：
- Name（必填）
- Proxy Type 下拉（SOCKS5 / SOCKS4 / HTTP CONNECT）
- Host（必填）
- Port（必填，根据类型自动填默认值）
- Username（可选）
- Password（可选）

### 4.2 ConnectModal — 新增 "Proxy" Tab

ConnectModal 三个 tab：**Authorization** | **SSH Tunnel** | **Proxy**

```
┌──────────────────────────────────────────────────────────┐
│  [Authorization]  [SSH Tunnel]  [Proxy]                  │
│                                                          │
│  ── Network Proxy ──────────────────────────────────────│
│                                                          │
│  Proxy   [None ▼]                                        │
│           ├── None (Direct)                              │
│           ├── Office (SOCKS5, proxy.corp:1080)           │
│           └── Home (HTTP, squid.local:8080)              │
│                                                          │
│  ── Connection Path ────────────────────────────────────│
│  Client → Office (SOCKS5) → Bastion → Target            │
│                                                          │
│  No proxy configured yet? Go to Settings → Proxies      │
└──────────────────────────────────────────────────────────┘
```

**交互规则**：
- 代理下拉从 proxies 表加载，显示 `name (type, host:port)`
- 默认 "None (Direct)" — 无代理直连
- 可与 SSH Tunnel tab 中的 bastion 组合使用
- Connection Path 动态合并显示代理 + bastion 链路
- 如果代理列表为空，显示提示 "Go to Settings → Proxies"

### 4.3 Sidebar 代理徽章

已配置网络代理的服务器在 Sidebar 中显示小徽章（类似 bastion 徽章），让用户快速识别。

## 五、凭证安全

代理密码与 SSH 密码采用相同的安全策略：
- 主存储：OS Keychain（`proxy_password_keychain_id` 字段）
- 回退：AES-256-GCM 加密（`proxy_password_enc` 字段）
- 数据库中永不存储明文密码
- 遵循现有 `keychain` 模块模式
