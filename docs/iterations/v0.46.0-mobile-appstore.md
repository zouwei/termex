## 一、触摸手势

### 1.1 双指缩放

终端内双指缩放调整字号：

- 检测 pinch gesture（`touchstart` + `touchmove` 双指距离变化）
- 实时调整 `terminal.options.fontSize`
- 松手后触发 `fitAddon.fit()` 重新计算列行数
- 字号范围: 8px - 24px
- 缩放过程中显示当前字号提示气泡

### 1.2 长按选择

终端内长按选择文本：

- 长按 500ms → 进入选择模式
- 显示选择手柄（iOS 风格）
- 选中文本后显示浮动操作栏：复制 / 粘贴 / 搜索
- 与 xterm.js 内置 selection 机制配合

### 1.3 滑动手势

| 手势 | 操作 |
|------|------|
| 左边缘右滑 | 返回服务器列表 |
| 左右滑动（多会话时） | 切换 SSH 会话标签 |
| 下拉（在服务器列表） | 刷新列表 |

## 二、iPad 布局实现

> v0.40.0 已在 `usePlatform.ts` 中预留 `isTablet` 检测，本迭代正式实现 iPad 专属布局。

### 2.1 iPad 侧边栏布局

屏幕宽度 >= 768px 判定为 tablet：

```
┌──────┬──────────────────┐
│      │                  │
│ 服务器│   终端/SFTP/设置  │
│ 列表  │                  │
│      │                  │
└──────┴──────────────────┘
```

- 左侧固定侧边栏（约 280px，非可拖拽）
- 右侧内容区（终端/SFTP/设置）
- 复用桌面 `Sidebar.vue`，加大触摸目标（44px 行高）
- 去掉鼠标拖拽 resize handle

**修改文件**: `src/components/mobile/MobileLayout.vue`

```vue
<template>
  <div v-if="isTablet" class="tablet-layout">
    <Sidebar class="tablet-sidebar" />
    <main class="tablet-content">
      <!-- 根据当前页面渲染 -->
    </main>
  </div>
  <div v-else class="phone-layout">
    <!-- 底部 Tab Bar 导航 -->
  </div>
</template>
```

### 2.2 Split View / Slide Over

- 响应 viewport 尺寸变化（`ResizeObserver` 已有）
- Split View 切换时终端自动重新 fit
- Slide Over 模式（约 320px 宽）降级为手机布局

### 2.3 外接键盘

iPad 连接外接键盘时：

| 快捷键 | 操作 |
|--------|------|
| Cmd+T | 新建连接 |
| Cmd+W | 断开当前会话 |
| Cmd+1-9 | 切换会话标签 |
| Cmd+, | 打开设置 |
| Cmd+K | 清除终端 |

通过 `attachCustomKeyEventHandler` 实现，与桌面端快捷键逻辑复用。

## 三、无障碍

### 3.1 VoiceOver (iOS) / TalkBack (Android)

| 区域 | 无障碍支持 |
|------|-----------|
| 服务器列表 | 每项添加 `aria-label`：服务器名称 + 连接状态 |
| 底部 Tab Bar | 每个 Tab 添加角色和标签 |
| 设置页 | 所有表单元素添加 `aria-label` |
| 终端 | 标记为 `role="application"`，屏幕阅读器直接传递到终端 |
| 操作菜单 | Bottom Sheet 添加焦点管理和 Escape 关闭 |

### 3.2 Dynamic Type (iOS)

响应 iOS 系统字号设置：

- 非终端区域（服务器列表、设置页）的文字大小跟随系统
- 使用 CSS `font-size: env(--apple-dynamic-type-body)` 或 JavaScript 检测
- 终端区域字号独立控制（由用户在设置中调整）

### 3.3 颜色对比度

确保所有 UI 元素满足 WCAG 2.1 AA 标准：
- 文字与背景对比度 >= 4.5:1
- 大文字 >= 3:1
- 交互元素焦点指示器可见

### 5.3 审核要点

**Apple 审核重点**:
- 应用必须能在审核人员设备上运行（提供测试服务器或演示模式）
- 不能仅是 WebView 包装（Tauri 使用原生框架，通常可通过）
- 需要解释为什么需要网络权限（SSH 连接）
- 隐私政策必须说明不收集用户数据

**Google Play 审核重点**:
- 需要声明 INTERNET 权限
- 如使用前台服务，需要声明用途
- 需要说明应用功能和目标用户
- 符合 Data Safety 声明要求
