# V0.17.0 — 服务器间 SFTP 文件传输

## Context

Termius 的核心差异化功能之一：允许用户在两台远程服务器之间直接传输文件，无需先下载到本地再上传。基于对现有 SFTP 架构的全面评估，本迭代将此能力引入 Termex。

### 当前状态

- SFTP 双栏布局硬编码为「左本地 + 右远程」
- `sftpStore` 为单会话模型（一个 sessionId、一个 currentPath、一个 entries）
- 传输方向仅支持 `upload | download`（本地↔远程）
- 后端 `SftpHandle` 的 `download`/`upload` 仅与本地文件系统交互

### 目标

1. SFTP 双栏中**两栏均可选择远程服务器**（不限于"左本地 + 右远程"）
2. 支持两台远程服务器间**拖拽/复制粘贴文件**传输
3. 传输进度实时显示，与现有本地↔远程传输共存
4. 向后兼容：默认行为不变（左本地 + 右远程）

---

## 一、技术方案

### 1.1 传输管道：32KB 流式中继

数据**必须经过本地中继**（两台服务器间通常无法直接连接 — 防火墙/NAT）：

```
Server A → [32KB SFTP read] → Termex 内存 → [32KB SFTP write] → Server B
```

- 恒定内存占用 ~128KB（2-4 个 in-flight 32KB chunk），支持任意大文件
- 排除方案：全文件内存加载（大文件 OOM）、临时文件中转（双倍 IO）

### 1.2 数据流

```
用户拖拽文件从 Server A 栏到 Server B 栏
  ↓
SftpFilePane drop handler 检测到 remote-to-remote
  ↓
sftpStore.serverToServerTransfer(sessionA, srcPath, sessionB, dstPath)
  ↓
tauriInvoke("sftp_transfer", { srcSessionId, srcPath, dstSessionId, dstPath })
  ↓
[Rust] sftp_transfer 命令：
  - Arc::clone 两个 SftpHandle
  - tokio::spawn 后台任务
  - 立即返回 transfer_id
  ↓
[后台任务]
  loop {
    read 32KB from Server A via SFTP
    write 32KB to Server B via SFTP
    emit sftp://progress/{transfer_id}
  }
  ↓
[前端] sftpStore.listenTransfer 响应式更新 TransferItem
  ↓
TransfersPanel 显示 "Server A → Server B" + 进度条
```

## 五、补充设计（评估后纳入）

### 5.1 useSftpPane composable 拆分边界

| composable / 组件 | 职责 |
|---|---|
| `useSftpPane(side)` | 目录导航（listDir/enterDir/goUp/breadcrumbs）、文件操作（mkdir/delete/rename/createFile/paste）、剪贴板（copy/cut）、路径编辑状态 |
| `useSftpDrag(side)` | 拖拽逻辑（dragStart/dragEnter/dragLeave/dragOver/drop）、MIME 类型判断、传输类型推断 |
| `SftpFilePane.vue` | 模式选择器 UI、工具栏、文件列表渲染、上下文菜单绑定、FileInfoDialog 挂载 |

### 5.2 多 SFTP 会话生命周期

- 切换栏从 Server A → Server B 时，**不主动关闭** Server A 的 SFTP 会话（可能另一栏还在用）
- SFTP 面板关闭时，关闭**所有由面板打开的 SFTP 会话**（维护 `openedByPanel: Set<string>` 追踪）
- SSH 断连时，若栏的 sessionId 匹配断连会话 → 自动回退到 Local 模式

### 5.3 同服务器不同目录拖拽

- 两栏 sessionId 相同 → 使用 `sftp_rename`（服务器内移动，零网络开销）
- 两栏 sessionId 不同 → 使用 `sftp_transfer`（跨服务器中继）
- 任一栏为 Local → 使用现有 `sftp_upload` / `sftp_download`

### 5.4 传输取消

- `AppState` 新增 `active_sftp_transfers: TokioRwLock<HashMap<String, oneshot::Sender<()>>>`
- `sftp_transfer` / `sftp_download` / `sftp_upload` 创建 oneshot channel，sender 存入 map
- 传输循环每次迭代 `try_recv` 检查取消信号
- 新增 `sftp_cancel_transfer(transfer_id)` 命令
- TransfersPanel 中每个进行中的传输显示取消按钮

### 5.5 v0.13.0 右键菜单完整迁移

SftpFilePane 需完整迁移 RemoteFilePane 的上下文菜单（14 项）：
- 主菜单：Download / Edit(TODO) / Copy / Cut / Paste / Rename / Delete
- More 子菜单：Copy Path / New File / New Folder / Select All(TODO) / Refresh / File Info / Edit Permissions(TODO)
- Local 模式下仅显示：Copy Path / Refresh
- FileInfoDialog 和 ChmodDialog 随面板迁移

### 5.6 目录递归传输（Phase 4 延后）

暂不实现，当用户拖拽目录时提示 "Directory transfer coming soon"。后续可通过递归 `list_dir` + 逐文件 `transfer_between` 实现。

## 六、风险评估

| # | 风险 | 影响 | 缓解措施 |
|---|------|------|----------|
| 1 | **sftpStore 重构范围大** | 所有读 `sftpStore.entries` 的组件需更新 | 保留向后兼容计算属性，渐进迁移 |
| 2 | **中继延迟** | 两端高速服务器间传输受本地带宽限制 | 无法绕过（NAT 限制），可调大 chunk 至 64-128KB |
| 3 | **传输中断** | 一端断连导致另一端残留不完整文件 | 捕获错误发 error 事件，UI 显示失败状态 |
| 4 | **800 行限制** | SftpFilePane 合并两个面板可能超限 | 提取 `useSftpPane` composable 分离逻辑 |
| 5 | **russh-sftp 并发安全** | 两个 SftpHandle 独立连接，无共享状态 | 低风险，已验证 Arc 模式 |
