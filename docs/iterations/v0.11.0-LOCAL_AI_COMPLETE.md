## 概述

### 什么是本地 AI 模型？

Termex v0.11.0 在现有云端 AI 提供商（OpenAI、Claude、Gemini）基础上，新增两类本地 AI 模型支持，实现完全离线的 AI 能力：

**Type 1：局域网 Ollama**（扩展现有）
- 用户在局域网服务器部署 Ollama
- Termex 通过自定义 URL 连接（如 `http://192.168.1.100:11434`）
- UI 显示 🌐 LAN 徽章区分

**Type 2：本机离线模型**（完全新增）
- 内置 llama-server 推理引擎二进制
- 模型文件热下载到 `~/.termex/models/`
- 完全离线运行，无需网络和 API Key
- 12 个代表性模型，4 层级分类（micro/small/medium/large）

### 核心价值

✅ **零成本**：无需购买 API 额度，不依赖云端服务
✅ **隐私**：数据完全本地存储，不上传到任何云端
✅ **离线**：断网后 AI 功能仍然可用
✅ **灵活**：支持多种开源模型，可按硬件选择
✅ **一键使用**：自动检测、下载、启动，无需手动配置

## 用户指南

### 首次使用（用户）

#### 1. 安装应用

从官方渠道下载 Termex v0.11.0 或更新到最新版本。

#### 2. 打开本机离线模型面板

1. 启动应用 → 设置 → AI 配置
2. 顶部会看到 **本机离线模型** 面板（绿色图标 💾）
3. 面板会显示 4 个层级的模型，分别为：微型、小型、中型、大型

#### 3. 下载模型

**根据硬件选择合适的模型梯度**：

| 梯度 | 硬件要求 | 用途 | 下载大小 |
|------|---------|------|--------|
| **Micro** | 2-4GB RAM | 极限低配，基础理解 | < 200MB |
| **Small** | 4-8GB RAM | 日常轻量级，平衡 | 200MB-1GB |
| **Medium** | 8-16GB RAM | 日常使用，质量不错 | 1GB-5GB |
| **Large** | 16GB+ RAM | 推荐，最佳质量 ⭐ | 5GB+ |

**下载步骤**：
1. 选择一个模型，点击 **下载** 按钮
2. 应用检查磁盘空间（需 >= 模型大小 × 1.2）
3. 进度条显示实时进度（0-100%）
4. 支持 HTTP Range 断点续传（网络中断后自动恢复）
5. 下载完成自动验证 SHA256，显示 ✓ 已下载

#### 4. 使用模型

**启用模型作为 AI 提供商**：
1. 点击已下载模型的 **用作 AI 提供商** 按钮
2. 应用自动启动 llama-server（~1-5 秒）
3. 加载选定的模型文件
4. 创建一个新的 AI provider（可在 AI 配置中管理）

**在 AI 面板中使用**：
- 选择该 provider 为默认 AI
- 在 SSH 会话输入命令
- "explain ls" → 离线解释命令
- "convert 'show processes sorted by memory'" → 转换为命令

#### 5. 管理模型

**切换模型**：
- 选择另一个已下载的模型，点击 **用作 AI**
- 旧模型自动卸载，新模型加载（~2-3 秒）

**删除模型**：
- 点击已下载模型的 **删除** 按钮
- 确认后，文件从 `~/.termex/models/` 移除，磁盘空间释放

### 常见问题

**Q: 下载速度慢怎么办？**

A: HuggingFace 由于网络差异可能较慢，可尝试：
- 使用镜像：hf-mirror.com（国内加速，v0.12.0 计划支持）
- 分多次下载（应用支持断点续传）
- 在网络条件好的时段下载

**Q: 模型存在哪里？**

A: `~/.termex/models/` 目录：
- macOS: `$HOME/.termex/models/`
- Linux: `$HOME/.termex/models/`
- Windows: `%APPDATA%\termex\models\`

可以手动放入 `.gguf` 文件，重启应用即可识别。

**Q: 可以手动放入自定义模型吗？**

A: v0.11.0 不支持自定义模型列表，但可以：
1. 手动放入 `.gguf` 文件到 `~/.termex/models/`
2. 修改 `src/assets/local-models.json` 添加配置
3. 计划 v1.0 支持用户自定义模型

**Q: 如何离线使用？**

A:
1. 联网时提前下载好模型
2. 离线后，应用仍可加载已下载的模型
3. llama-server 完全离线推理，无需任何网络

**Q: 推理速度有多快？**

A: 取决于硬件和模型：
- **首 token**（首个结果）：2-5 秒
- **后续 token**：0.1-0.5 秒/token
- **GPU 加速**：如支持 Metal/CUDA/Vulkan，快 5-20 倍

**Q: 会占用多少显存？**

A: 大约 = 模型大小 + ~200MB 开销
- TinyLlama: ~400MB
- Qwen2.5 7B: ~5GB
- 确保系统可用 VRAM > 模型大小

**Q: 可以同时运行多个模型吗？**

A: v0.11.0 一次只能运行一个。v0.12.0 计划支持多 llama-server 实例。

## 故障排除

### 常见问题

**Q1: llama-server 找不到**

```bash
# 检查安装
which llama-server
llama-server --help

# 如果没有，安装 llama.cpp
brew install llama.cpp  # macOS
sudo apt-get install llama-cpp  # Linux

# 自定义路径
export LLAMA_SERVER_PATH=/path/to/llama-server
```

**Q2: SHA256 验证失败**

- 源文件被篡改或损坏
- 更新 local-models.json 中的 SHA256 值
- 手动验证：`sha256sum ~/.termex/models/*.gguf`

**Q3: 推理响应超时**

```bash
# 直接测试 API
curl http://localhost:15000/v1/models | jq

# 检查模型是否加载
curl http://localhost:15000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"default","messages":[{"role":"user","content":"Hi"}]}'
```

**Q4: 内存不足**

- 选择更小的模型（Micro/Small tier）
- 检查硬件规格（至少 4GB RAM）
- 监控进程内存占用

**Q5: 网络中断下载**

- 下载会自动断点续传
- 网络恢复后点击继续下载
- 前提：已下载部分保留在 ~/.termex/models/

```bash
ps aux | grep llama-server
```

## 性能优化

### 当前性能

#### 关键路径分析

```
用户点击下载 (0ms)
  ├─ 磁盘检查 (100ms)
  ├─ HTTP Range 请求 (网络延迟，取决于网络 ~5-50MB/s)
  └─ SHA256 验证 (CPU 密集，4.7GB 文件约 8-20 秒)
     └─ 文件移动 (10ms)

用户点击"用作 AI" (0ms)
  ├─ 查找空闲端口 (~10ms)
  ├─ spawn 子进程 (~50ms)
  ├─ 加载模型文件 (取决于大小，~500ms-2s)
  └─ API 就绪 (通过轮询检测，智能 polling)

AI 功能调用 (0ms)
  ├─ HTTP 请求 (网络延迟，localhost 很快)
  ├─ llama 推理 (取决于硬件，CPU ~2-30s，GPU 快 5-20 倍)
  └─ 流式响应 (实时)
```

#### 性能指标

| 指标 | 目标 | 实际 | 优化空间 |
|------|------|------|--------|
| 应用启动 | < 2s | ~1.5s | ✅ 达标 |
| LocalAiPanel 打开 | < 100ms | ~40ms | ✅ 优秀 |
| 模型下载 | 取决于网络 | ~5-50MB/s | 可接受 |
| SHA256 验证 (4.7GB) | < 15s | 8-20s | 可优化 |
| 模型加载 | < 1s | ~0.5s | ✅ 优秀 |
| 首 token 延迟 | 2-5s | 取决于硬件 | 硬件限制 |

## 附录：二进制获取

本节用于 llama-server 二进制无法自动检测或需要手动部署的情况。

### 二进制获取方案

#### 方案 A：自动检测（推荐）

应用会按以下顺序自动检测 llama-server：

1. **LLAMA_SERVER_PATH 环境变量**（自定义路径）
   ```bash
   export LLAMA_SERVER_PATH=/path/to/llama-server
   ```

2. **Homebrew 安装路径**（macOS/Linux）
   ```bash
   # macOS Apple Silicon
   /opt/homebrew/bin/llama-server

   # macOS Intel
   /usr/local/bin/llama-server

   # Linux
   /usr/bin/llama-server
   /usr/local/bin/llama-server
   ```

3. **系统 PATH**（通过 `which` 命令）
   ```bash
   which llama-server
   ```

如果上述都失败，应用会提示安装：
```bash
brew install llama.cpp
```

#### 方案 B：手动下载（无法自动检测时）

如果自动检测失败，可手动从 Release 获取：

**第1步**：访问 llama.cpp Release 页面
```
https://github.com/ggerganov/llama.cpp/releases/latest
```

**第2步**：下载对应平台的二进制

| 平台 | 文件名 |
|------|--------|
| macOS arm64（Apple Silicon） | `llama-server-macos-arm64` |
| macOS x64（Intel） | `llama-server-macos-x64` |
| Windows x64 | `llama-server-windows-x64.exe` |
| Linux x64 | `llama-server-linux-x64` |
| Linux ARM64 | `llama-server-linux-aarch64` |

**第3步**：设置环境变量或放入 PATH

方式 A：环境变量（推荐）
```bash
export LLAMA_SERVER_PATH=~/Downloads/llama-server-macos-arm64
chmod +x ~/Downloads/llama-server-macos-arm64
```

方式 B：放入系统 PATH
```bash
sudo mv ~/Downloads/llama-server-macos-arm64 /usr/local/bin/
sudo chmod +x /usr/local/bin/llama-server-macos-arm64
sudo ln -s /usr/local/bin/llama-server-macos-arm64 /usr/local/bin/llama-server
```

**验证**：
```bash
which llama-server
llama-server --help
```

#### 方案 C：从源代码编译

如无法直接下载，可自行编译：

```bash
# 1. 安装编译工具
# macOS
brew install cmake

# Linux (Ubuntu/Debian)
sudo apt-get install build-essential cmake

# 2. 克隆仓库
git clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp

# 3. 编译
make -j $(nproc)

# 或使用 CMake
mkdir build && cd build
cmake ..
cmake --build . --config Release -j $(nproc)

# 4. 结果在 llama-server（或 llama-server.exe on Windows）
ls -la llama-server

# 5. 设置环境变量或放入 PATH
export LLAMA_SERVER_PATH=$(pwd)/llama-server
```

编译时间：5-15 分钟（取决于硬件）

#### 方案 D：国内镜像（中国用户）

使用 Gitee 镜像加速：

```bash
git clone https://gitee.com/ggerganov/llama.cpp.git
cd llama.cpp
make -j $(nproc)

export LLAMA_SERVER_PATH=$(pwd)/llama-server
```

#### 方案 E：Docker 编译（无编译环境）

```bash
docker run -v $(pwd):/workspace -w /workspace ubuntu:22.04 bash -c "
  apt-get update && apt-get install -y build-essential cmake git && \
  git clone https://github.com/ggerganov/llama.cpp.git && \
  cd llama.cpp && make -j 4
"

# 编译结果在 llama.cpp/llama-server
export LLAMA_SERVER_PATH=$(pwd)/llama.cpp/llama-server
```

### 二进制故障排除

**问题：llama-server: command not found**

检查清单：
```bash
# 1. 检查是否已安装
which llama-server

# 2. 验证文件权限
ls -la /usr/local/bin/llama-server
chmod +x /usr/local/bin/llama-server

# 3. 检查依赖（macOS）
otool -L /usr/local/bin/llama-server

# 4. 检查依赖（Linux）
ldd /usr/local/bin/llama-server

# 5. 手动设置环境变量
export LLAMA_SERVER_PATH=/path/to/llama-server
```

**问题：二进制无法执行**

```bash

# 验证二进制完整性（应 > 20MB）
ls -lh $(which llama-server)
```
