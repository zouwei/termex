# v0.44.0 — Mobile AI + Native Keychain + Performance

> 状态：**计划中**

## 背景

v0.42.0 完成了移动端的跨设备同步、服务器管理、SFTP 和设置。本迭代将云端 AI 功能带到移动端，接入原生凭据存储（iOS Keychain / Android Keystore），并优化性能和后台行为。

### 目标

1. 移动端云端 AI 功能（命令解释、NL2Cmd、智能补全）
2. iOS Keychain Services 原生凭据存储
3. Android Keystore 原生凭据存储
4. 终端性能优化（延迟 < 32ms）
5. 后台保活与前台恢复重连

## 二、iOS Keychain Services

### 2.1 方案

使用 `security-framework` Rust crate（Security.framework 的纯 Rust 绑定）接入 iOS Keychain。

**修改文件**: `src-tauri/src/keychain/mod.rs`

```rust
#[cfg(target_os = "ios")]
mod ios_keychain {
    use security_framework::passwords::*;
    // 实现与桌面端 keyring 相同的接口
    // 单条目 JSON 模式: service="termex", account="__termex_store__"
}
```

### 2.2 行为一致性

| 行为 | 桌面端 (keyring) | iOS (security-framework) |
|------|-----------------|-------------------------|
| 首次启动提示 | 系统 keychain 授权弹窗 | 无提示（应用级 keychain） |
| 存储模式 | 单条目 JSON | 单条目 JSON |
| 加密 | 系统 keychain 加密 | iOS Keychain 加密 |
| 回退 | AES-256-GCM | 无需回退 |

### 2.3 依赖

```toml
[target.'cfg(target_os = "ios")'.dependencies]
security-framework = "2"
```

## 四、后台保活

### 4.1 iOS 后台行为

iOS 应用切后台后约 30 秒被挂起：
- SSH 连接通过 TCP keepalive 尽量维持
- 恢复前台时检测连接状态
- 如已断开 → 显示重连 UI → 自动重连
- 重连成功 → 恢复终端会话

### 4.2 Android 后台行为

Android 允许更长的后台执行：
- 默认行为: 应用切后台保持连接
- 可选: 前台服务通知（用户在设置中 opt-in）→ 长时间保持连接
- 电池优化白名单提示

### 4.4 Android 权限声明

`AndroidManifest.xml` 中需要声明的权限：

| 权限 | 用途 | 必需 |
|------|------|------|
| `INTERNET` | SSH/SFTP/AI API 网络连接 | 是 |
| `ACCESS_NETWORK_STATE` | 检测网络状态变化（断线重连） | 是 |
| `FOREGROUND_SERVICE` | 后台保持 SSH 连接（用户 opt-in） | 可选 |
| `FOREGROUND_SERVICE_DATA_SYNC` | 前台服务类型声明 (Android 14+) | 可选 |
| `POST_NOTIFICATIONS` | 前台服务通知 (Android 13+) | 可选 |
| `WAKE_LOCK` | 防止 CPU 休眠断开连接 | 可选 |

前台服务需要创建 `NotificationChannel`（Android 8+），在 Kotlin 侧实现。

### 4.3 实现

**新建文件**: `src/composables/useMobileLifecycle.ts`

```typescript
// 监听 Tauri app lifecycle 事件
// 'tauri://focus' — 应用恢复前台
// 'tauri://blur' — 应用切到后台
// 检测 SSH 连接状态，必要时重连
```

## 五、性能优化

### 5.1 目标

| 指标 | 目标值 | 测试设备 |
|------|--------|---------|
| 终端输入延迟 | < 32ms | iPhone 13+, Pixel 7+ |
| 单会话内存 | < 80MB | — |
| Canvas 渲染帧率 | >= 30fps | — |
| 应用冷启动 | < 3s | — |

### 5.2 优化措施

| 优化项 | 说明 |
|--------|------|
| 滚动缓冲区减半 | 移动端默认 scrollback 从 5000 降至 2500 行 |
| 后台暂停 IPC | 应用切后台时暂停非必要的 Tauri 事件轮询 |
| 终端字体渲染 | 确保系统字体匹配，避免额外字体下载 |
| 内存泄漏检测 | Profile 多次连接/断开是否有未释放资源 |
