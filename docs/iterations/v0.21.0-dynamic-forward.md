# V0.21.0 — SSH Dynamic Port Forwarding (SOCKS5 Proxy)

## Context

远程 Linux 服务器无 GUI/浏览器，用户需要借助本地浏览器 + 远程服务器的网络出口，实现"在远程环境中浏览网页"的效果。这是 `ssh -D <port>` 的经典用法 — 在本地启动 SOCKS5 代理，所有流量通过 SSH 隧道从远程服务器出口。

### 当前状态

- v0.3.0 定义了 Local / Remote / Dynamic 三种转发类型
- **Local Forward 已完整实现**：`TcpListener` → `channel_open_direct_tcpip()` → `bridge_streams()`
- Remote Forward / Dynamic Forward：DB schema + 前端类型已定义，**后端未实现**
- `port_forward_start` 命令只处理 local，dynamic 类型调用会失败
- 多层代理 + 跳板机链路不影响（动态转发工作在 SSH 会话层，传输层透明）

### 设计决策

- Dynamic Forward 复用已有 `direct-tcpip` 通道机制，唯一区别是目标地址从 SOCKS5 协议动态解析
- SOCKS5 服务端只实现 CONNECT 命令（浏览器代理场景），不实现 BIND / UDP ASSOCIATE
- 监听 `127.0.0.1` 默认无需认证（本机访问安全），预留认证扩展接口
- 支持 DOMAINNAME 地址类型（DNS 透传到远程解析，避免 DNS 泄露）
- Remote Forward 本迭代不实现（v0.20.0 已升级 russh 0.49 并用于 Git Sync 反向转发，通用 Remote Forward UI 留待后续）

## 一、SOCKS5 协议实现

### 1.1 协议范围（RFC 1928 子集）

| 特性 | 实现 | 说明 |
|------|------|------|
| Method Negotiation | ✅ | 仅支持 NO AUTH (0x00) |
| CONNECT 命令 | ✅ | 核心功能 |
| BIND 命令 | ❌ | 浏览器不使用 |
| UDP ASSOCIATE | ❌ | 浏览器不使用 |
| IPv4 地址 | ✅ | 0x01 |
| DOMAINNAME 地址 | ✅ | 0x03，DNS 在远程解析 |
| IPv6 地址 | ✅ | 0x04 |

### 1.2 协议流程

```
浏览器                    SOCKS5 Server (Termex)              SSH Channel
  │                              │                                │
  │── [05 01 00] ──────────────→│  Method negotiation             │
  │←── [05 00] ─────────────────│  NO AUTH selected               │
  │                              │                                │
  │── [05 01 00 03 len host port]→│  CONNECT request              │
  │                              │── channel_open_direct_tcpip() →│
  │                              │←── channel opened ─────────────│
  │←── [05 00 00 01 0 0 0 0 0 0]│  CONNECT success               │
  │                              │                                │
  │←─────── bridge_streams() ──────── bidirectional data ────────→│
```

### 1.3 Rust 实现

新增文件：`src-tauri/src/ssh/socks5.rs` (~150 行)

```rust
/// SOCKS5 address types.
enum Socks5Addr {
    IPv4([u8; 4], u16),
    Domain(String, u16),
    IPv6([u8; 16], u16),
}

/// Handles SOCKS5 handshake on an accepted TCP stream.
/// Returns the target (host, port) requested by the client.
pub async fn socks5_handshake(
    stream: &mut tokio::net::TcpStream,
) -> Result<(String, u16), Socks5Error> {
    // 1. Read method negotiation → reply NO AUTH
    // 2. Read CONNECT request → parse address type
    // 3. Return (host, port) for caller to open SSH channel
    // 4. Caller sends success/failure reply after channel open
}

/// Sends SOCKS5 CONNECT success reply.
pub async fn socks5_reply_success(stream: &mut tokio::net::TcpStream) -> io::Result<()>;

/// Sends SOCKS5 CONNECT failure reply.
pub async fn socks5_reply_failure(stream: &mut tokio::net::TcpStream, code: u8) -> io::Result<()>;
```

### 1.4 错误码

| 代码 | 含义 | 触发场景 |
|------|------|---------|
| 0x00 | 成功 | SSH channel 打开成功 |
| 0x01 | General failure | SSH channel 打开失败 |
| 0x04 | Host unreachable | 远程无法连接目标 |
| 0x05 | Connection refused | 远程目标拒绝连接 |
| 0x07 | Command not supported | 非 CONNECT 命令 |

## 二、forward.rs 扩展

### 2.1 新增 `start_dynamic_forward()`

在 `src-tauri/src/ssh/forward.rs` 中新增，复用已有 `bridge_streams()`：

```rust
/// Starts a dynamic port forward (`-D`): listens on `local_host:local_port`,
/// runs a SOCKS5 server, and tunnels each connection via SSH direct-tcpip.
pub async fn start_dynamic_forward(
    app: tauri::AppHandle,
    session_id: String,
    forward_id: String,
    local_host: String,
    local_port: u16,
    registry: &ForwardRegistry,
) -> Result<(), SshError> {
    // 1. TcpListener::bind(local_host:local_port)
    // 2. accept loop (same pattern as start_local_forward)
    // 3. Per connection:
    //    a. socks5_handshake() → get (target_host, target_port)
    //    b. channel_open_direct_tcpip(target_host, target_port)
    //    c. socks5_reply_success()
    //    d. bridge_streams()
}
```

### 2.2 连接处理流程对比

```
Local Forward:
  accept → channel_open_direct_tcpip(fixed_host, fixed_port) → bridge_streams()

Dynamic Forward:
  accept → socks5_handshake() → channel_open_direct_tcpip(parsed_host, parsed_port) → bridge_streams()
```

差异仅在目标地址的来源：配置固定 vs SOCKS5 协议动态解析。

## 三、命令层修改

### 3.1 `port_forward_start` 扩展

`src-tauri/src/commands/port_forward.rs`：

```rust
#[tauri::command]
pub async fn port_forward_start(
    state: State<'_, AppState>,
    app: AppHandle,
    session_id: String,
    forward_id: String,
    forward_type: String,        // 新增参数
    local_host: String,
    local_port: u16,
    remote_host: Option<String>,  // dynamic 时为 None
    remote_port: Option<u16>,     // dynamic 时为 None
) -> Result<(), String> {
    match forward_type.as_str() {
        "local" => {
            forward::start_local_forward(
                app, session_id, forward_id,
                local_host, local_port,
                remote_host.unwrap_or("127.0.0.1".into()),
                remote_port.unwrap_or(80),
                &state.forwards,
            ).await
        }
        "dynamic" => {
            forward::start_dynamic_forward(
                app, session_id, forward_id,
                local_host, local_port,
                &state.forwards,
            ).await
        }
        _ => Err(SshError::Other(format!("Unsupported forward type: {}", forward_type))),
    }
    .map_err(|e| e.to_string())
}
```

### 3.2 前端 Store 适配

`src/stores/portForwardStore.ts` — `startForward()` 传入 `forwardType`：

```typescript
async function startForward(sessionId: string, forward: PortForward): Promise<void> {
  await tauriInvoke("port_forward_start", {
    sessionId,
    forwardId: forward.id,
    forwardType: forward.forwardType,   // 新增
    localHost: forward.localHost,
    localPort: forward.localPort,
    remoteHost: forward.remoteHost,      // dynamic 时为 null
    remotePort: forward.remotePort,      // dynamic 时为 null
  });
  activeForwards.value.add(forward.id);
}
```

### 4.4 Auto Start

DB 已有 `auto_start` 字段。SSH 连接成功后（`openShell` 之后），
前端遍历该服务器的转发规则，对 `auto_start = true` 的规则自动调用 `port_forward_start`。

### 4.5 `port_forward_start` 命令改造

当前命令不传 `forward_type`。改造方案：**从 DB 读取**，不改前端签名。

```rust
// port_forward_start 内部：
// 1. 从 DB 查询 forward_id 对应的 forward_type
// 2. match forward_type 路由到 start_local_forward / start_dynamic_forward
// 前端仅传 session_id + forward_id，无需传 type/host/port（全从 DB 读）
```

这简化了前端调用，避免前后端参数不一致。

## 十、安全考量

| 风险 | 措施 |
|------|------|
| 非本机访问 SOCKS5 代理 | 默认绑定 `127.0.0.1`，UI 警告非 loopback 地址 |
| DNS 泄露 | 支持 DOMAINNAME 地址类型，DNS 在远程解析；UI 提示开启 Proxy DNS |
| 端口冲突 | 启动前检测端口占用，明确报错 |
| SSH 断连 | channel 失败时向浏览器返回 SOCKS5 错误码，不静默挂起 |
